# Quick Start Guide

Download, install, and run your first query.

## Step 1: Download TablePro

The fastest way is via [Homebrew](https://brew.sh):

```bash
brew install --cask tablepro
```

<Tip>
If you installed via Homebrew, skip to [Step 3](#step-3-create-your-first-connection). The cask formula handles installation and removes the macOS quarantine attribute automatically.
</Tip>

Alternatively, download manually:

1. Visit the [GitHub Releases page](https://github.com/datlechin/tablepro/releases)
2. Download the DMG file for your Mac:
   - **Apple Silicon** (M1/M2/M3/M4/M5): `TablePro-arm64.dmg`
   - **Intel**: `TablePro-x86_64.dmg`

<Note>
Not sure which chip your Mac has? Click the Apple menu and select **About This Mac**. Look for "Chip" (Apple Silicon) or "Processor" (Intel).
</Note>

## Step 2: Install TablePro

1. Open the downloaded DMG file
2. Drag **TablePro** to your **Applications** folder
3. Eject the DMG
4. Open TablePro from your Applications folder

<Warning>
On first launch, macOS shows a **"TablePro is damaged and can't be opened"** error because the app is not signed with an Apple Developer certificate. Fix it by opening **Terminal** and running:

```bash
xattr -cr /Applications/TablePro.app
```

Then open TablePro again. One-time step. See the [Installation guide](/installation#first-launch-security) for details.
</Warning>

## Step 3: Create Your First Connection

The Welcome screen appears on launch. Create a new database connection from here.

### For MySQL/MariaDB

1. Click **New Connection**
2. Select **MySQL** or **MariaDB** as the database type
3. Fill in the connection details:
   - **Name**: A friendly name for this connection (e.g., "Local MySQL")
   - **Host**: `localhost` (or your server address)
   - **Port**: `3306` (default)
   - **Username**: Your database username (e.g., `root`)
   - **Password**: Your database password
   - **Database**: (Optional) Default database to connect to

### For PostgreSQL

1. Click **New Connection**
2. Select **PostgreSQL** as the database type
3. Fill in the connection details:
   - **Name**: A friendly name (e.g., "Local PostgreSQL")
   - **Host**: `localhost` (or your server address)
   - **Port**: `5432` (default)
   - **Username**: Your database username (e.g., `postgres`)
   - **Password**: Your database password
   - **Database**: The database name to connect to

### For Amazon Redshift

1. Click **New Connection**
2. Select **Amazon Redshift** as the database type
3. Fill in the connection details:
   - **Name**: A friendly name (e.g., "Redshift Warehouse")
   - **Host**: Your cluster endpoint (e.g., `my-cluster.abc123.us-east-1.redshift.amazonaws.com`)
   - **Port**: `5439` (default)
   - **Username**: Your Redshift username (e.g., `awsuser`)
   - **Password**: Your password
   - **Database**: The database name (usually `dev`)

### For SQLite

1. Click **New Connection**
2. Select **SQLite** as the database type
3. Click **Browse** to select your SQLite database file
4. Give your connection a name

<Tip>
SQLite databases don't require authentication. Just select the `.sqlite` or `.db` file on your Mac.
</Tip>

### For MongoDB

1. Click **New Connection**
2. Select **MongoDB** as the database type
3. Fill in the connection details:
   - **Name**: A friendly name (e.g., "Local MongoDB")
   - **Host**: `localhost` (or your server address)
   - **Port**: `27017` (default)
   - **Username**: Your database username (if authentication is enabled)
   - **Password**: Your database password
   - **Database**: The database name to connect to
   - **Auth Database**: Authentication database (defaults to `admin`)

<Tip>
For local MongoDB instances without authentication, you only need the host and port. Connections use the `mongodb://` URI protocol internally.
</Tip>

## Step 4: Test and Connect

1. Click **Test Connection** to verify your settings
2. A green checkmark confirms the connection works
3. Click **Create** to save and connect

## Step 5: Explore Your Database

The main interface has three panels:

- **Sidebar**: Your databases and tables
- **SQL Editor**: Write and execute SQL queries
- **Data Grid**: View and edit query results

### Try Your First Query

1. Click on a table in the sidebar to view its contents
2. Or type a query in the SQL editor:

```sql
SELECT * FROM users LIMIT 10;
```

3. Press `Cmd+Enter` to execute the query
4. View the results in the data grid below
