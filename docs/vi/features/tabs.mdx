# Tab Truy Vấn

Mỗi tab là một workspace độc lập với SQL, kết quả, phân trang, sắp xếp và filter riêng. Mở bao nhiêu tùy thích. Tất cả tồn tại qua các lần khởi động lại ứng dụng.

## Tổng Quan

Tab là workspace chính trong TablePro. Mỗi kết nối nhớ tập tab riêng, và tab tồn tại qua các lần khởi động lại nên bạn tiếp tục đúng chỗ đã dừng.

## Loại Tab

TablePro có hai loại tab:

| Loại | Biểu Tượng | Mục Đích |
|------|------|---------|
| **Tab Truy Vấn** | Biểu tượng tài liệu | Viết và chạy SQL tùy chỉnh |
| **Tab Bảng** | Biểu tượng bảng (xanh) | Duyệt dữ liệu bảng với phân trang và lọc |

### Tab Truy Vấn

Tab truy vấn là SQL editor đa năng. Dùng cho query ad-hoc, khám phá dữ liệu với SELECT tùy chỉnh, chạy DDL/DML, và test query trước khi đưa vào ứng dụng.

Tab truy vấn bắt đầu trống. Gõ SQL và nhấn `Cmd+Enter` để chạy.

### Tab Bảng

Tab bảng mở khi click tên bảng trong sidebar:

- Tự động tạo query `SELECT * FROM table LIMIT n`
- Phân trang phía server với điều khiển chuyển trang
- Chỉnh sửa inline với theo dõi thay đổi
- Chuyển đổi xem Structure để xem/sửa schema bảng
- Hỗ trợ filter để thu hẹp kết quả

<Tip>
Tab bảng cho phép chỉnh sửa mặc định. Tab truy vấn chỉ đọc trừ khi query là `SELECT * FROM table` đơn giản.
</Tip>

### Tái Sử Dụng Tab Thông Minh

Khi click bảng trong sidebar:

- **Cùng bảng**: Chuyển sang tab hiện có (không chạy query mới)
- **Bảng khác**: Mở tab mới

Bật **Tái sử dụng tab bảng sạch** trong **Cài đặt** > **Tab** để có hành vi giống TablePlus: click bảng khác sẽ thay thế tab bảng hiện tại nếu không có thay đổi chưa lưu, không có tương tác người dùng (sắp xếp, lọc, v.v.) và không bị ghim. Giữ thanh tab gọn mà vẫn bảo vệ công việc.

## Quản Lý Tab

### Tạo Tab

| Hành Động | Cách Thực Hiện |
|--------|-----|
| Tab truy vấn mới | Click nút **+** trong thanh tab, hoặc nhấn `Cmd+T` |
| Tab truy vấn mới (từ toolbar) | Click nút **SQL** trên toolbar |
| Tab bảng mới | Click tên bảng trong sidebar |
| Tab bảng mới (với query) | Mở tab truy vấn, rồi dùng SQL editor |

### Đóng Tab

| Hành Động | Cách Thực Hiện |
|--------|-----|
| Đóng tab hiện tại | Nhấn `Cmd+W` |
| Đóng tab cụ thể | Di chuột qua tab, click nút **x** |
| Đóng từ menu ngữ cảnh | Click phải vào tab, chọn **Close Tab** |
| Đóng các tab khác | Click phải vào tab, chọn **Close Other Tabs** |

<Warning>
Đóng tab có thay đổi chưa lưu sẽ mất thay đổi đó. TablePro cảnh báo trước khi đóng nếu có sửa đổi dữ liệu đang chờ.
</Warning>

### Chuyển Đổi Tab

Click bất kỳ tab nào, hoặc dùng phím tắt:

- `Cmd+1` đến `Cmd+9` nhảy đến tab theo vị trí
- `Cmd+Shift+[` hoặc `Cmd+Option+Left` cho tab trước
- `Cmd+Shift+]` hoặc `Cmd+Option+Right` cho tab tiếp theo

Trạng thái tab trước được bảo toàn hoàn toàn:

- Nội dung SQL editor và vị trí con trỏ
- Kết quả query
- Vị trí cuộn trong data grid
- Hàng đã chọn
- Trạng thái sắp xếp và filter
- Thay đổi đang chờ

### Sắp Xếp Lại Tab

Kéo tab đến vị trí mới. Chỉ báo chèn màu xanh hiển thị nơi tab sẽ đặt.

Thanh tab cuộn ngang khi có nhiều tab hơn không gian hiển thị. Tab đang chọn tự động cuộn vào vùng nhìn thấy.

### Ghim Tab

Ghim tab quan trọng để tránh đóng nhầm:

1. Click phải vào tab
2. Chọn **Pin Tab**
3. Biểu tượng ghim xuất hiện trên tab

Tab đã ghim:

- Không thể đóng qua nút đóng, `Cmd+W`, hay menu ngữ cảnh
- Không bị đóng khi chọn **Close Other Tabs**
- Không bị thay thế khi click bảng khác trong sidebar
- Vẫn mở qua các phiên

Bỏ ghim: click phải, chọn **Unpin Tab**.

### Nhân Bản Tab

Tạo bản sao của tab:

1. Click phải vào tab
2. Chọn **Duplicate Tab**
3. Tab mới mở với cùng nội dung query và kết quả

Tab nhân bản được chèn ngay sau tab gốc.

## Trạng Thái Tab

Mỗi tab lưu toàn bộ trạng thái, được bảo toàn khi chuyển tab.

### Thuộc Tính Trạng Thái

| Thuộc Tính | Mô Tả |
|----------|-------------|
| **Tab ID** | UUID duy nhất |
| **Title** | Tên hiển thị trong thanh tab |
| **SQL Content** | Văn bản query trong SQL editor |
| **Tab Type** | Truy vấn hoặc bảng |
| **Table Name** | Tên bảng liên quan (cho tab bảng) |
| **Result Columns** | Tên cột từ query cuối |
| **Result Rows** | Dữ liệu hàng từ query cuối (cache) |
| **Column Types** | Metadata kiểu dữ liệu |
| **Execution Time** | Thời lượng query cuối |
| **Error Message** | Lỗi từ query thất bại cuối |
| **Is Editable** | Chỉnh sửa inline có bật không |
| **Pagination State** | Trang hiện tại, kích thước trang, tổng hàng |
| **Sort State** | Cột sắp xếp và hướng |
| **Filter State** | Filter đang hoạt động |
| **Pending Changes** | Sửa đổi dữ liệu đang chờ |
| **Selected Rows** | Chỉ số hàng đang chọn |
| **Result Version** | Bộ đếm cache invalidation |

### Theo Dõi Thay Đổi Theo Tab

Mỗi tab có thay đổi đang chờ riêng. Khi chuyển tab:

1. Thay đổi của tab hiện tại lưu vào thuộc tính `pendingChanges`
2. Change manager tải thay đổi đang chờ của tab mới
3. Chỉ báo trực quan cập nhật theo trạng thái tab mới

Bạn có thể có chỉnh sửa đang chờ ở nhiều tab cùng lúc mà không xung đột.

## Tính Liên Tục Tab

Tab tồn tại qua các lần khởi động lại ứng dụng.

### Những Gì Được Lưu

| Được Lưu | Không Được Lưu |
|-------|-----------|
| Tab ID và tiêu đề | Kết quả query (truy vấn lại khi mở) |
| Nội dung SQL | Thay đổi dữ liệu đang chờ |
| Loại tab | Hàng đã chọn |
| Tên bảng | Trạng thái sắp xếp và filter |
| Trạng thái ghim | Thời gian thực thi |

### Cách Hoạt Động

<Steps>
  <Step title="Tự Động Lưu Với Debounce">
    Trạng thái tab tự động lưu 500ms sau thay đổi (debounce để tránh ghi liên tiếp)
  </Step>
  <Step title="Lưu Theo Kết Nối">
    Mỗi kết nối database có trạng thái tab riêng, lưu trong UserDefaults với key theo kết nối
  </Step>
  <Step title="Khôi Phục Khi Kết Nối">
    Khi kết nối lại database, TablePro khôi phục tab đã lưu và chạy lại query cho tab bảng
  </Step>
  <Step title="Xử Lý Đóng Cửa Sổ">
    Khi đóng cửa sổ, lưu debounce đang chờ được flush ngay để tránh mất dữ liệu
  </Step>
</Steps>

### Nhớ Query Cuối Cùng

TablePro cũng nhớ query cuối cho mỗi kết nối, tương tự TablePlus. Dù đóng hết tab, query cuối vẫn được giữ và khôi phục khi tạo tab mới.

## Phân Trang Trong Tab

TablePro dùng chiến lược phân trang khác nhau theo loại tab.

### Tab Bảng (Phân Trang Phía Server)

Tab bảng dùng SQL `LIMIT` và `OFFSET`:

- Chỉ tải trang hiện tại từ database
- Chuyển trang kích hoạt query mới
- Kích thước trang mặc định cấu hình trong **Settings** > **Data Grid**

| Điều Khiển | Hành Động |
|---------|--------|
| Trang đầu | Nhảy đến trang 1 |
| Trang trước | Lùi một trang |
| Chỉ báo trang | Hiện trang hiện tại / tổng trang |
| Trang tiếp | Tiến một trang |
| Trang cuối | Nhảy đến trang cuối |

<Tip>
Phân trang server-side tiết kiệm bộ nhớ. Chỉ trang đang xem được giữ trong RAM, dù bảng có hàng triệu hàng.
</Tip>

### Tab Truy Vấn (Phân Trang Phía Client)

Tab truy vấn chạy query đầy đủ và nhận tất cả kết quả. Với tập kết quả lớn, TablePro phân trang client-side:

- Kết quả lưu trong bộ nhớ
- Phân trang cắt mảng kết quả mà không truy vấn lại
- Sắp xếp thực hiện cục bộ trên kết quả cache

### Cấu Hình Kích Thước Trang

Cấu hình kích thước trang mặc định trong **Settings** > **Data Grid**:

| Tùy Chọn | Hàng Mỗi Trang |
|--------|---------------|
| Nhỏ | 100 |
| Trung bình | 500 |
| Lớn | 1,000 |
| Tùy chỉnh | 10 - 100,000 |

## Thực Hành Tốt Nhất

### Tổ Chức Theo Tác Vụ

Dùng tab riêng cho từng tác vụ: một tab khám phá dữ liệu, một tab viết migration, một tab test. Giữ workspace có tổ chức.

### Ghim Tab Quan Trọng

Ghim tab chứa query hay dùng. Tránh đóng nhầm.

### Dùng Tab Bảng Để Duyệt

Khi chỉ cần xem dữ liệu, click bảng trong sidebar mở tab bảng. Có sẵn phân trang, chỉnh sửa và xem cấu trúc mà không cần viết SQL.

### Dùng Tab Truy Vấn Cho Việc Phức Tạp

Cho JOIN, aggregation, subquery và thao tác nhiều câu lệnh, dùng tab truy vấn. SQL editor cho toàn quyền kiểm soát.

### Đóng Tab Không Dùng

Tab nhẹ, nhưng đóng tab không cần nữa giữ thanh tab gọn và dễ tìm hơn.
