# Theo Dõi Thay Đổi

Thay đổi trong TablePro được xếp hàng trong bộ nhớ, không áp dụng ngay. Chỉnh sửa ô, thêm hàng, xóa hàng, rồi xem lại tất cả trước khi commit. Không gì chạm đến database cho đến khi bạn xác nhận.

## Tổng Quan

Thay đổi dữ liệu hoặc cấu trúc bảng được xếp hàng cục bộ và đánh dấu trong grid. Commit tất cả cùng lúc hoặc hủy bỏ hoàn toàn. Cách này tránh mất dữ liệu do tai nạn và cho bạn toàn quyền kiểm soát thời điểm thay đổi đến database.

<Tip>
Theo dõi thay đổi hoạt động theo tab. Mỗi tab có tập thay đổi đang chờ riêng, nên chuyển tab không ảnh hưởng đến chỉnh sửa.
</Tip>

## Thay Đổi Dữ Liệu

TablePro theo dõi ba loại thay đổi: chỉnh sửa ô (update), thêm hàng (insert) và xóa hàng (delete). Tra cứu đều O(1) nhờ cấu trúc dữ liệu hash, nên tốc độ vẫn nhanh dù có nhiều thay đổi đang chờ.

### Chỉnh Sửa Ô

Để chỉnh sửa ô:

1. Double-click vào ô cần sửa
2. Nhập giá trị mới
3. Nhấn `Enter` để xác nhận hoặc `Escape` để hủy

Ô đã sửa được đánh dấu trong data grid.

</Note>

### Thêm Hàng

Để thêm hàng mới:

1. Click nút **+** trên toolbar hoặc dùng phím tắt
2. Hàng mới xuất hiện cuối data grid, đánh dấu bằng chỉ báo chèn
3. Điền giá trị cho từng cột
4. Cột có giá trị mặc định được điền sẵn `DEFAULT`

Hàng mới được theo dõi riêng biệt với update. Chỉnh sửa ô trong hàng vừa thêm được gộp vào phép chèn, không tạo bản ghi update riêng.

### Xóa Hàng

Để xóa hàng:

1. Chọn một hoặc nhiều hàng trong data grid
2. Nhấn `Delete` hoặc click nút xóa
3. Hàng bị xóa được đánh dấu bằng chỉ báo gạch ngang

Hàng đã xóa vẫn hiện trong grid cho đến khi commit hoặc hủy, để bạn có thể undo nếu cần.

## Commit & Hủy Bỏ

Sau khi xếp hàng các thay đổi, commit vào database hoặc hủy bỏ.

### Commit Thay Đổi

Để áp dụng tất cả thay đổi đang chờ:

1. Click nút **Commit** trên toolbar, hoặc nhấn `Cmd+S`
2. TablePro tạo câu lệnh SQL có tham số cho tất cả thay đổi trong hàng đợi
3. Câu lệnh chạy trên database
4. Thành công thì hàng đợi xóa sạch và grid làm mới

Commit tạo ra:

| Loại Thay Đổi | SQL Được Tạo |
|-------------|--------------|
| Chỉnh sửa ô | `UPDATE ... SET column = ? WHERE pk = ?` |
| Thêm hàng | `INSERT INTO ... (columns) VALUES (?)` |
| Xóa hàng | `DELETE FROM ... WHERE pk = ?` |

<Note>
UPDATE yêu cầu primary key trên bảng. Nếu không có primary key, TablePro hiển thị lỗi khi commit update. DELETE có thể hoạt động không cần primary key bằng cách khớp tất cả giá trị cột.
</Note>

### Hủy Bỏ Thay Đổi

Để hoàn nguyên tất cả thay đổi đang chờ:

1. Click nút **Discard** trên toolbar
2. Tất cả thay đổi trong hàng đợi bị xóa
3. Ô đã sửa quay về giá trị ban đầu
4. Hàng đã thêm bị xóa khỏi grid
5. Hàng đã xóa được bỏ đánh dấu

Hủy bỏ cũng xóa stack undo/redo cho tab hiện tại.

### Xem Trước SQL Dữ Liệu

Trước khi commit, xem trước SQL sẽ chạy:

1. Thay đổi dữ liệu trong grid (sửa ô, thêm/xóa hàng)
2. Click nút **Preview SQL** (biểu tượng mắt) trên toolbar, hoặc nhấn `Cmd+Shift+P`
3. Popover hiển thị tất cả SQL đang chờ với syntax highlighting
4. Dùng **Copy All** để sao chép SQL vào clipboard

## Hoàn Tác & Làm Lại

Stack undo/redo đầy đủ cho chỉnh sửa dữ liệu. Bước lùi và tiến qua các thay đổi trước khi commit.

### Cách Hoạt Động

Mọi thay đổi dữ liệu được đẩy vào stack undo:

- **Chỉnh sửa ô**: Lưu giá trị cũ và mới
- **Thêm hàng**: Lưu chỉ số hàng
- **Xóa hàng**: Lưu chỉ số hàng và dữ liệu gốc
- **Xóa hàng loạt**: Lưu tất cả hàng bị ảnh hưởng thành một hành động

Undo chuyển hành động sang stack redo. Redo chuyển ngược lại.

### Sử Dụng Hoàn Tác & Làm Lại

| Hành Động | Phím Tắt |
|--------|----------|
| Hoàn tác | `Cmd+Z` |
| Làm lại | `Cmd+Shift+Z` |

<Tip>
Undo/redo nhận biết ngữ cảnh. Khi SQL editor được focus, `Cmd+Z` undo chỉnh sửa văn bản. Khi data grid được focus, nó undo thay đổi dữ liệu.
</Tip>

### Hành Vi Stack

- Stack undo/redo được duy trì theo tab
- Commit hoặc hủy bỏ xóa cả hai stack
- Thay đổi mới sau undo xóa stack redo (hành vi undo tiêu chuẩn)

## Thay Đổi Schema

TablePro cũng theo dõi thay đổi cấu trúc bảng (cột, index, foreign key) theo cách tương tự.

### Các Thao Tác Schema Được Theo Dõi

| Thao Tác | Những Gì Được Theo Dõi |
|-----------|----------------|
| **Thêm cột** | Định nghĩa cột mới (tên, kiểu, nullable, default, v.v.) |
| **Sửa đổi cột** | Định nghĩa cũ và mới |
| **Xóa cột** | Cột đánh dấu để xóa |
| **Thêm index** | Định nghĩa index mới (tên, cột, kiểu, tính duy nhất) |
| **Sửa đổi index** | Định nghĩa cũ và mới |
| **Xóa index** | Index đánh dấu để xóa |
| **Thêm foreign key** | Định nghĩa FK mới (cột, tham chiếu, hành động) |
| **Sửa đổi foreign key** | Định nghĩa cũ và mới |
| **Xóa foreign key** | FK đánh dấu để xóa |
| **Sửa đổi primary key** | Cột primary key cũ và mới |

### Chỉ Báo Trực Quan

Trong tab Structure, thay đổi đang chờ hiển thị bằng chỉ báo trực quan:

- **Mục mới** (cột, index, foreign key) đánh dấu bằng màu chèn
- **Mục đã sửa** hiển thị trường đã thay đổi
- **Mục đã xóa** đánh dấu bằng chỉ báo xóa

Thay đổi schema có stack undo/redo riêng (tách biệt với thay đổi dữ liệu):

- Tối đa 100 mức undo
- Hành động mới xóa stack redo
- Hỗ trợ tất cả loại thao tác schema

## Tạo SQL

TablePro tạo SQL theo từng loại database cho cả thay đổi dữ liệu và schema.

### SQL Dữ Liệu (`SQLStatementGenerator`)

Thay đổi dữ liệu tạo câu lệnh có tham số để ngăn SQL injection:

<Tabs>
  <Tab title="MySQL/MariaDB">
    ```sql
    -- UPDATE với LIMIT 1 để an toàn
    UPDATE `users` SET `name` = ? WHERE `id` = ? LIMIT 1

    -- INSERT
    INSERT INTO `users` (`name`, `email`) VALUES (?, ?)

    -- DELETE (với PK)
    DELETE FROM `users` WHERE `id` = ? OR `id` = ?

    -- DELETE (không có PK, khớp tất cả cột)
    DELETE FROM `users` WHERE `name` = ? AND `email` = ? LIMIT 1
    ```
  </Tab>
  <Tab title="PostgreSQL">
    ```sql
    -- UPDATE với tham số vị trí
    UPDATE "users" SET "name" = $1 WHERE "id" = $2

    -- INSERT
    INSERT INTO "users" ("name", "email") VALUES ($1, $2)

    -- DELETE (với PK)
    DELETE FROM "users" WHERE "id" = $1 OR "id" = $2
    ```
  </Tab>
  <Tab title="SQLite">
    ```sql
    -- UPDATE
    UPDATE "users" SET "name" = ? WHERE "id" = ?

    -- INSERT
    INSERT INTO "users" ("name", "email") VALUES (?, ?)

    -- DELETE
    DELETE FROM "users" WHERE "id" = ? OR "id" = ?
    ```
  </Tab>
</Tabs>

### SQL Schema (`SchemaStatementGenerator`)

Thay đổi schema tạo câu lệnh ALTER TABLE với cú pháp theo từng database:

<Tabs>
  <Tab title="MySQL/MariaDB">
    ```sql
    -- Thêm cột
    ALTER TABLE `users` ADD COLUMN `phone` VARCHAR(20) NOT NULL

    -- Sửa đổi cột
    ALTER TABLE `users` MODIFY COLUMN `name` VARCHAR(200) NOT NULL

    -- Xóa cột
    ALTER TABLE `users` DROP COLUMN `phone`

    -- Thêm index
    CREATE INDEX `idx_email` ON `users` (`email`) USING BTREE

    -- Xóa foreign key
    ALTER TABLE `orders` DROP FOREIGN KEY `fk_orders_user`
    ```
  </Tab>
  <Tab title="PostgreSQL">
    ```sql
    -- Thêm cột
    ALTER TABLE "users" ADD COLUMN "phone" VARCHAR(20) NOT NULL

    -- Sửa đổi cột (nhiều câu lệnh)
    ALTER TABLE "users" RENAME COLUMN "name" TO "full_name";
    ALTER TABLE "users" ALTER COLUMN "full_name" TYPE VARCHAR(200);
    ALTER TABLE "users" ALTER COLUMN "full_name" SET NOT NULL

    -- Xóa cột
    ALTER TABLE "users" DROP COLUMN "phone"

    -- Thêm index
    CREATE INDEX "idx_email" ON "users" ("email")

    -- Xóa constraint
    ALTER TABLE "orders" DROP CONSTRAINT "fk_orders_user"
    ```
  </Tab>
  <Tab title="SQLite">
    ```sql
    -- Thêm cột
    ALTER TABLE "users" ADD COLUMN "phone" TEXT

    -- Xóa cột
    ALTER TABLE "users" DROP COLUMN "phone"

    -- Thêm index
    CREATE INDEX "idx_email" ON "users" ("email")
    ```

    <Warning>
    SQLite hỗ trợ ALTER TABLE hạn chế. Sửa đổi cột và xóa foreign key cần tái tạo bảng, hiện chưa hỗ trợ trong trình chỉnh sửa trực quan. Dùng SQL editor cho các thao tác này.
    </Warning>
  </Tab>
</Tabs>

### Hỗ Trợ Hàm SQL

Khi chỉnh sửa ô, TablePro nhận ra các hàm SQL phổ biến và chèn dạng literal thay vì chuỗi được trích dẫn:

| Hàm | Database |
|----------|----------|
| `NOW()` | MySQL, MariaDB |
| `CURRENT_TIMESTAMP` | Tất cả |
| `CURDATE()` | MySQL, MariaDB |
| `CURRENT_DATE` | Tất cả |
| `SYSDATE()` | MySQL |
| `UTC_TIMESTAMP()` | MySQL, MariaDB |

### Sắp Xếp Theo Phụ Thuộc

Thay đổi schema tự động sắp xếp theo thứ tự phụ thuộc trước khi chạy:

1. Xóa foreign key (phải trước để tránh vi phạm ràng buộc)
2. Xóa index
3. Xóa/sửa cột
4. Thêm cột
5. Sửa primary key
6. Thêm index
7. Thêm foreign key

## Thực Hành Tốt Nhất

### Xem Xét Trước Khi Commit

Kiểm tra thay đổi đang chờ trước khi commit. Chỉ báo trực quan trong data grid giúp phát hiện chỉnh sửa không mong muốn.

### Sử Dụng Hoàn Tác Thường Xuyên

Stack undo xóa khi commit, nên hãy tận dụng nó trong khi chỉnh sửa. Undo/redo không tốn chi phí gì.

### Commit Thường Xuyên

Tránh tích lũy quá nhiều thay đổi. Commit nhỏ, thường xuyên dễ kiểm tra hơn và ít rủi ro hơn.

### Xem Trước Thay Đổi Schema

Luôn dùng SQL preview khi sửa cấu trúc bảng. Kiểm tra chính xác câu lệnh ALTER TABLE trước khi chạy.

### Sao Lưu Trước Thay Đổi Phá Hủy

Khi xóa cột hoặc thay đổi schema phá hủy khác, sao lưu bảng trước:

```sql
CREATE TABLE users_backup AS SELECT * FROM users;
```
