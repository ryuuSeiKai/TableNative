# AI Chat

TablePro tích hợp bảng chat AI để viết, giải thích và sửa câu truy vấn SQL. Hỗ trợ Claude (Anthropic), OpenAI, Gemini (Google), OpenRouter, Ollama và các endpoint tương thích OpenAI.

## Mở bảng AI Chat

| Phương thức | Thao tác |
|-------------|----------|
| Phím tắt | `Cmd+Shift+L` |
| Thanh công cụ | Nhấp nút AI (biểu tượng sparkles) |
| Menu | **View** > **Toggle AI Chat** |

Bảng chat hiện ở bên phải trình soạn thảo, cạnh data inspector.

## Trò chuyện với AI

1. Nhập câu hỏi vào ô nhập ở cuối
2. Nhấn **Enter** hoặc nhấp nút gửi
3. AI phản hồi dựa trên ngữ cảnh kết nối database hiện tại

AI tự động nhận:
- Loại database (MySQL, PostgreSQL, SQLite)
- Schema (bảng, cột, kiểu dữ liệu, khóa ngoại)
- Câu truy vấn hiện tại trong editor (khi bật)

<Tip>
AI tạo SQL đúng dialect database của bạn, dùng đúng cú pháp và dấu ngoặc kép định danh.
</Tip>

### Phản hồi Markdown

Phản hồi AI hiển thị đầy đủ Markdown:
- **Văn bản định dạng** -- tiêu đề, in đậm, in nghiêng, danh sách
- **Khối code SQL** -- tô sáng cú pháp và nút thao tác
- **Code nội tuyến** -- cho tên bảng, tên cột, từ khóa SQL

## Khối code

Khối code SQL trong phản hồi AI có hai nút:

- **Copy** -- sao chép SQL vào clipboard
- **Insert to Editor** -- chèn SQL vào tab editor đang hoạt động

## Token Usage

Sau mỗi phản hồi, số token hiển thị bên dưới tin nhắn assistant:

- **Input tokens** -- token gửi đến AI (tin nhắn + ngữ cảnh)
- **Output tokens** -- token AI tạo ra

Dùng để theo dõi lượng sử dụng API và chi phí mỗi tin nhắn.

## Lịch sử hội thoại

Hội thoại tự động lưu vào đĩa và khôi phục khi khởi động lại. Quản lý từ tiêu đề bảng chat:

| Nút | Thao tác |
|-----|----------|
| **Lịch sử** (biểu tượng đồng hồ) | Duyệt và chuyển giữa các hội thoại |
| **Chat mới** (biểu tượng bút chì) | Tạo hội thoại mới |
| **Xóa** (biểu tượng thùng rác) | Xóa hội thoại hiện tại |

Lưu dưới dạng JSON trong `~/Library/Application Support/TablePro/ai_chats/`.

<Tip>
Hội thoại được tự động đặt tiêu đề từ tin nhắn đầu tiên, dễ tìm lại sau.
</Tip>

## Thử lại và Tái tạo

- **Retry** -- nếu tạo phản hồi thất bại (lỗi mạng, rate limit, v.v.), nút "Retry" hiện bên dưới tin nhắn cuối. Nhấp để gửi lại.
- **Regenerate** -- sau phản hồi thành công, nút "Regenerate" hiện bên dưới tin nhắn assistant. Nhấp để xóa và tạo phản hồi mới.

## Streaming

Phản hồi stream theo thời gian thực:
- Chỉ báo ba chấm hiển thị khi chờ token đầu tiên
- Nhấp **Stop** để hủy bất cứ lúc nào
- Cuộn lên đọc tin nhắn trước mà không bị kéo về cuối. Auto-scroll chỉ kích hoạt khi đang xem nội dung mới nhất.
- Có thể gửi tin nhắn mới khi đang xem phản hồi cũ

## Các nhà cung cấp được hỗ trợ

Sáu loại nhà cung cấp:

| Nhà cung cấp | Endpoint | Yêu cầu API Key |
|--------------|----------|-----------------|
| **Claude** (Anthropic) | `https://api.anthropic.com` | Có |
| **OpenAI** | `https://api.openai.com` | Có |
| **OpenRouter** | `https://openrouter.ai/api` | Có |
| **Gemini** (Google) | `https://generativelanguage.googleapis.com` | Có |
| **Ollama** | `http://localhost:11434` | Không |
| **Custom** | Bất kỳ endpoint tương thích OpenAI | Tùy chọn |

<Tip>
**Tự động phát hiện Ollama**: Nếu Ollama đang chạy khi TablePro khởi động, nó tự động đăng ký làm nhà cung cấp AI với model đầu tiên khả dụng. Không cần cấu hình thủ công.
</Tip>

<Note>
Có thể cấu hình nhiều nhà cung cấp cùng lúc và định tuyến từng tính năng AI đến nhà cung cấp khác nhau.
</Note>

## Cấu hình nhà cung cấp AI

Thiết lập trong **Settings** (`Cmd+,`) > **AI**:

### Thêm nhà cung cấp

1. Nhấp **Add Provider** trong phần Providers
2. Chọn loại (Claude, OpenAI, Gemini, OpenRouter, Ollama, hoặc Custom)
3. Nhập API key (endpoint điền sẵn theo loại)
4. Chỉ định model, hoặc nhấp **Fetch** để lấy danh sách model. Model tự động tải khi thay đổi cài đặt.
5. Nhấp **Test** để kiểm tra kết nối

### Bảo mật API Key

API key lưu trong macOS Keychain. Không bao giờ lưu dạng text thuần hay trong file cấu hình.

### Định tuyến tính năng

Định tuyến từng tính năng AI đến nhà cung cấp khác nhau:

| Tính năng | Mô tả |
|-----------|-------|
| **Chat** | Trò chuyện AI tổng quát |
| **Explain Query** | Giải thích câu truy vấn |
| **Optimize Query** | Đề xuất tối ưu hiệu năng |
| **Fix Error** | Chẩn đoán và đề xuất sửa lỗi |
| **Inline Suggestions** | Gợi ý code nội tuyến |

Mỗi tính năng có thể dùng nhà cung cấp riêng hoặc fallback về mặc định (nhà cung cấp đầu tiên được bật).

## Cài đặt ngữ cảnh

Kiểm soát ngữ cảnh database gửi đến AI:

| Cài đặt | Mặc định | Mô tả |
|---------|----------|-------|
| **Include database schema** | Bật | Gửi định nghĩa bảng và cột |
| **Include current query** | Bật | Gửi query từ tab editor đang hoạt động |
| **Include query results** | Tắt | Gửi kết quả truy vấn gần đây |
| **Max schema tables** | 20 | Số bảng tối đa trong ngữ cảnh |

## Chính sách AI theo kết nối

Kiểm soát quyền truy cập AI theo từng kết nối:

| Chính sách | Mô tả |
|------------|-------|
| **Use Default** | Theo chính sách mặc định từ cài đặt AI |
| **Always Allow** | Luôn cho phép AI cho kết nối này |
| **Ask Each Time** | Hỏi trước khi gửi dữ liệu đến AI |
| **Never** | Tắt hoàn toàn AI cho kết nối này |

Hữu ích để khóa AI trên kết nối production.

### Phím tắt

| Thao tác | Phím tắt | Mô tả |
|----------|----------|-------|
| Bật/tắt AI Chat | `Cmd+Shift+L` | Mở hoặc đóng bảng AI chat |
| Giải thích với AI | `Cmd+L` | Giải thích SQL đã chọn (hoặc query hiện tại) |
| Tối ưu với AI | `Cmd+Option+L` | Tối ưu SQL đã chọn (hoặc query hiện tại) |

Tất cả phím tắt AI tùy chỉnh được trong **Settings** > **Keyboard** ở mục **AI**.

<Tip>
Khi không chọn văn bản nào, `Cmd+L` và `Cmd+Option+L` dùng toàn bộ query từ tab editor đang hoạt động.
</Tip>

### Yêu cầu AI sửa lỗi

Khi query thất bại, hộp thoại lỗi có nút **Ask AI to Fix**:

1. Mở bảng AI chat (nếu chưa mở)
2. Gửi query lỗi và thông báo lỗi đến AI
3. AI phân tích và đề xuất query đã sửa
