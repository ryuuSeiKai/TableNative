# SQL Autocomplete

Autocomplete phân tích vị trí con trỏ, ngữ cảnh cú pháp SQL, bảng và alias được tham chiếu, cùng schema database để chỉ gợi ý những gì phù hợp tại mỗi điểm trong query.

## Cách Autocomplete Hoạt Động

Engine autocomplete phân tích bốn yếu tố:

1. **Vị trí hiện tại** trong query
2. **Ngữ cảnh cú pháp SQL** (SELECT, FROM, WHERE, v.v.)
3. **Bảng được tham chiếu** và alias
4. **Schema database** (bảng, cột, hàm)

Gợi ý được lọc theo những gì có nghĩa tại vị trí con trỏ.

## Kích Hoạt Autocomplete

### Tự động

Autocomplete xuất hiện tự động khi gõ:

- Sau 1-2 ký tự
- Sau `.` (dấu chấm) để truy cập cột
- Sau một số từ khóa (SELECT, FROM, JOIN, v.v.)

### Thủ công

Bắt buộc hiển thị popup:

- Bắt đầu gõ và đợi một chút
- Popup xuất hiện với gợi ý phù hợp

### Đóng

- `Escape` để đóng popup
- Tiếp tục gõ để lọc
- Click ngoài popup

## Các Loại Completion

### Từ Khóa SQL

Từ khóa được gợi ý theo ngữ cảnh:

```sql
SEL|  -- Gợi ý: SELECT
FROM users WH|  -- Gợi ý: WHERE
SELECT * FROM users WHERE name LIKE '%test%' ORD|  -- Gợi ý: ORDER
```

**Từ khóa được gợi ý**:

| Danh mục | Từ khóa |
|----------|----------|
| Câu lệnh | SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, DROP |

| Toán tử | AND, OR, NOT, IN, LIKE, BETWEEN, IS NULL, EXISTS |
| Join | INNER JOIN, LEFT JOIN, RIGHT JOIN, CROSS JOIN |

### Tên Bảng

Bảng được gợi ý sau từ khóa mong đợi tham chiếu bảng:

```sql
SELECT * FROM |  -- Gợi ý tất cả bảng
SELECT * FROM us|  -- Gợi ý bảng bắt đầu "us": users, user_roles
SELECT * FROM users JOIN |  -- Gợi ý tất cả bảng
```

### Tên Cột

Cột được gợi ý theo bảng tham chiếu:

```sql
SELECT | FROM users  -- Gợi ý cột từ bảng users
SELECT u.| FROM users u  -- Gợi ý cột với prefix "u."
SELECT * FROM users WHERE |  -- Gợi ý cột từ users
```

#### Sau Table Alias

Gõ alias rồi `.` hiển thị cột của bảng đó:

```sql
SELECT
    u.|  -- Hiển thị: id, name, email, created_at (từ users)
FROM users u
JOIN orders o ON u.id = o.|  -- Hiển thị: id, user_id, total (từ orders)
```

### Hàm

Hàm SQL được gợi ý trong ngữ cảnh phù hợp:

```sql
SELECT |  -- Gợi ý: COUNT, SUM, AVG, MAX, MIN, v.v.
SELECT COUNT(|  -- Gợi ý cột và *
WHERE date_column > |  -- Gợi ý: NOW(), CURRENT_DATE, v.v.
```

**Hàm thường gặp**:

| Danh mục | Hàm |
|----------|-----------|
| Tổng hợp | COUNT, SUM, AVG, MAX, MIN |
| Chuỗi | CONCAT, SUBSTRING, UPPER, LOWER, TRIM |
| Ngày | NOW, CURRENT_DATE, DATE_FORMAT, DATEADD |
| Toán học | ROUND, ABS, CEIL, FLOOR |
| Điều kiện | COALESCE, NULLIF, CASE, IF |

### Tên Schema

Với database nhiều schema (PostgreSQL):

```sql
SELECT * FROM |  -- Gợi ý: public, schema1, schema2
SELECT * FROM public.|  -- Gợi ý bảng trong schema public
```

## Gợi Ý Theo Ngữ Cảnh

### Sau SELECT

Ngay sau SELECT:

- Tên cột từ bảng trong FROM
- `*` cho tất cả cột
- `DISTINCT`
- Hàm tổng hợp

```sql
SELECT |
-- Gợi ý: *, DISTINCT, tên cột, hàm
```

### Sau FROM/JOIN

Sau FROM hoặc JOIN:

- Tất cả tên bảng truy cập được
- Tên có schema (schema.table)

### Sau WHERE

Trong mệnh đề WHERE:

- Tên cột từ bảng đã chọn
- Toán tử so sánh
- Toán tử logic (AND, OR)

### Sau ON (điều kiện JOIN)

Trong JOIN ON:

- Cột từ cả hai bảng join
- Thường dùng cho quan hệ foreign key

```sql
SELECT * FROM users u
JOIN orders o ON |
-- Gợi ý: u.id, o.user_id, v.v.
```

## Sử Dụng Completion

### Điều Hướng Bàn Phím

| Phím | Hành động |
|-----|--------|
| `Down` | Gợi ý tiếp theo |
| `Up` | Gợi ý trước |
| `Enter` | Chấp nhận gợi ý |
| `Tab` | Chấp nhận gợi ý |
| `Escape` | Đóng popup |

### Lọc

Tiếp tục gõ để lọc:

```sql
SELECT * FROM user|
-- Hiển thị: users, user_roles, user_preferences
-- Gõ thêm thu hẹp kết quả
```

### Chấp Nhận Gợi Ý

Khi chấp nhận gợi ý:

1. Text được chèn tại con trỏ
2. Prefix đã gõ được thay thế
3. Con trỏ di chuyển đến cuối text chèn

## Cache Schema

### Cách Schema Được Tải

Khi kết nối, TablePro:

1. Lấy tất cả tên bảng
2. Tải thông tin cột cho mỗi bảng
3. Cache để autocomplete nhanh

### Làm Mới Schema

Nếu thay đổi schema (thêm bảng, đổi cột):

1. Click chuột phải kết nối trong sidebar
2. Chọn **Refresh**

Hoặc ngắt kết nối và kết nối lại.

<Tip>
Thay đổi schema trong TablePro (qua tab Structure) tự động phản ánh trong autocomplete.
</Tip>

## Hiệu Suất

### File Lớn

Autocomplete hoạt động trên file SQL bất kỳ kích thước, kể cả dump nhiều megabyte. Với file trên 500 KB, engine trích xuất cửa sổ cục bộ (~10 KB) quanh con trỏ và chỉ phân tích câu lệnh gần nhất. Gợi ý vẫn xuất hiện tức thì bất kể kích thước file.

### Schema Lớn

Với database trên 100 bảng:

- Tải schema ban đầu có thể mất chút thời gian
- Autocomplete vẫn nhanh sau khi cache
- Gõ thêm ký tự để thu hẹp nhanh hơn

### Mẹo Hiệu Suất

1. **Gõ thêm ký tự**: Thu hẹp gợi ý nhanh hơn
2. **Dùng table alias**: Giảm phạm vi tìm kiếm
3. **Để cache hoạt động**: Schema được cache sau lần tải đầu

## Giới Hạn

### Không Được Gợi Ý

- Bảng tạm (không trong schema)
- Alias subquery
- Tên cột động
- Cột từ kết nối khác

### Độ Phức Tạp Query

Query rất phức tạp có khả năng nhận biết ngữ cảnh hạn chế:

- Subquery lồng sâu
- CTE phức tạp
- SQL động

Gợi ý từ khóa và bảng cơ bản vẫn hoạt động trong các trường hợp này.

## Cấu Hình

Autocomplete bật mặc định, không cần cấu hình. Debounce 50ms áp dụng cho trigger completion để giảm công việc không cần thiết.

## Thực Hành Tốt

### Dùng Table Alias

Alias giúp autocomplete chính xác hơn:

```sql
-- Tốt: Rõ ràng cột từ bảng nào
SELECT u.name, o.total
FROM users u
JOIN orders o ON u.id = o.user_id

-- Kém rõ ràng hơn
SELECT users.name, orders.total
FROM users
JOIN orders ON users.id = orders.user_id
```

### Bắt Đầu Với FROM

Viết FROM trước cho autocomplete biết bảng:

```sql
-- Cách này giúp autocomplete
FROM users u
JOIN orders o ON u.id = o.user_id
SELECT u.|  -- Giờ đã biết bảng u và o
```

### Để Gợi Ý Hướng Dẫn

Không chắc tên cột? Thử:

1. Gõ alias và dấu chấm: `u.`
2. Duyệt gợi ý
3. Chọn cột đúng

## Khắc Phục Sự Cố

### Không Xuất Hiện Gợi Ý

1. **Kiểm tra kết nối**: Đảm bảo đã kết nối database
2. **Làm mới schema**: Chuột phải kết nối > Refresh
3. **Đợi cache**: Schema lớn cần thời gian tải

### Gợi Ý Sai

1. **Kiểm tra alias**: Đảm bảo alias đúng
2. **Kiểm tra FROM**: Bảng phải được tham chiếu để gợi ý cột
3. **Refresh khi schema đổi**: Thay đổi schema cần refresh

### Gợi Ý Chậm

1. **Gõ thêm ký tự**: Thu hẹp phạm vi tìm kiếm
2. **Dùng alias**: Giới hạn scope tìm cột
3. **Kiểm tra kết nối**: Độ trễ mạng ảnh hưởng tải schema
