## Truy Cập Cấu Trúc Bảng

Để xem cấu trúc bảng:

<Steps>
  <Step title="Chọn Bảng">
    Click tên bảng trong sidebar
  </Step>
  <Step title="Mở Tab Structure">
    Click tab **Structure** (bên dưới data grid)
  </Step>
  <Step title="Khám Phá">
    Điều hướng giữa các tab Columns, Indexes, Foreign Keys và DDL
  </Step>
</Steps>

Hoặc chuột phải vào bảng và chọn **Show Structure**.

|----------|-------------|
| **Name** | Tên cột |

| **SET DEFAULT** | Đặt thành giá trị mặc định |
| **NO ACTION** | Tương tự RESTRICT (kiểm tra hoãn lại) |

### Sao Chép DDL

1. Mở tab DDL
2. Chọn tất cả (`Cmd+A`)
3. Sao chép (`Cmd+C`)

Dùng DDL cho:

- Tạo lại bảng trong database khác
- Version control schema
- Tài liệu
- Tạo bảng tương tự

### Ví Dụ DDL

<Tabs>
  <Tab title="MySQL">
    ```sql
    CREATE TABLE `users` (
      `id` int NOT NULL AUTO_INCREMENT,
      `email` varchar(255) NOT NULL,
      `name` varchar(100) DEFAULT NULL,
      `created_at` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
      `updated_at` timestamp NULL DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP,
      PRIMARY KEY (`id`),
      UNIQUE KEY `email` (`email`),
      KEY `created_at` (`created_at`)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
    ```
  </Tab>
  <Tab title="PostgreSQL">
    ```sql
    CREATE TABLE users (
      id SERIAL PRIMARY KEY,
      email VARCHAR(255) NOT NULL UNIQUE,
      name VARCHAR(100),
      created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
      updated_at TIMESTAMP WITH TIME ZONE
    );

    CREATE INDEX idx_users_created_at ON users(created_at);
    ```
  </Tab>
  <Tab title="SQLite">
    ```sql
    CREATE TABLE users (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      email TEXT NOT NULL UNIQUE,
      name TEXT,
      created_at TEXT DEFAULT CURRENT_TIMESTAMP,
      updated_at TEXT
    );

    CREATE INDEX idx_users_created_at ON users(created_at);
    ```
  </Tab>
</Tabs>

## Sửa Đổi Cấu Trúc

<Warning>
Sửa đổi cấu trúc thay đổi schema database. Luôn backup dữ liệu quan trọng trước khi thực hiện.
</Warning>

### Editor Cấu Trúc Trực Quan

Sửa cấu trúc bảng trực quan, không cần SQL:

#### Thêm Cột

1. Mở **Structure** > **Columns**
2. Click **+** để thêm cột
3. Đặt thuộc tính: tên, kiểu, nullable, mặc định, auto-increment, comment
   - Click ô **Type** để mở popover chọn kiểu. Duyệt theo danh mục hoặc tìm kiếm theo tên.
   - Kiểu có tham số (ví dụ: `VARCHAR(255)`, `DECIMAL(10,2)`) nhập trực tiếp vào ô freeform
4. Click **Apply** để xem trước và thực thi

#### Sửa Đổi Cột

1. Click cột trong tab Columns
2. Chỉnh sửa thuộc tính trong panel chi tiết
3. Thay đổi được theo dõi với undo/redo
4. Click **Apply** để xem SQL ALTER TABLE

#### Xóa Cột

1. Chọn cột cần xóa
2. Click **-** hoặc phím Delete
3. Xác nhận
4. Apply

#### Bộ Chọn Kiểu

Khi sửa **Type**, popover tìm kiếm xuất hiện với kiểu dữ liệu theo database, tổ chức theo danh mục:

| Danh mục | Ví dụ |
|----------|-------|
| **Số** | INT, BIGINT, DECIMAL, FLOAT, DOUBLE |
| **Chuỗi** | VARCHAR, TEXT, CHAR, ENUM |
| **Ngày & Giờ** | DATE, DATETIME, TIMESTAMP, TIME |
| **Nhị phân** | BLOB, BINARY, BYTEA |
| **Khác** | JSON, JSONB, UUID, BOOLEAN |

Kiểu khả dụng theo database đang kết nối (MySQL, PostgreSQL, SQLite). Nhập trực tiếp kiểu tùy chỉnh cho kiểu có tham số như `VARCHAR(255)` hoặc `NUMERIC(10,2)`.

### Thêm Cột qua SQL

Thêm cột trực tiếp bằng SQL:

```sql
-- MySQL
ALTER TABLE users ADD COLUMN phone VARCHAR(20) AFTER email;

-- PostgreSQL
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

-- SQLite (ALTER TABLE bị giới hạn)
ALTER TABLE users ADD COLUMN phone TEXT;
```

### Sửa Đổi Cột qua SQL

```sql
-- MySQL
ALTER TABLE users MODIFY COLUMN name VARCHAR(200) NOT NULL;

-- PostgreSQL
ALTER TABLE users ALTER COLUMN name TYPE VARCHAR(200);
ALTER TABLE users ALTER COLUMN name SET NOT NULL;

-- SQLite (yêu cầu tạo lại bảng)
-- Xem tài liệu SQLite
```

### Xóa Cột qua SQL

```sql
-- MySQL/PostgreSQL
ALTER TABLE users DROP COLUMN phone;

-- SQLite (yêu cầu tạo lại bảng)
```

### Quản Lý Index

```sql
-- Tạo index
CREATE INDEX idx_users_email ON users(email);

-- Tạo unique index
CREATE UNIQUE INDEX idx_users_username ON users(username);

-- Xóa index
DROP INDEX idx_users_email ON users;  -- MySQL
DROP INDEX idx_users_email;  -- PostgreSQL
```

### Quản Lý Foreign Key

```sql
-- Thêm foreign key
ALTER TABLE orders
ADD CONSTRAINT fk_orders_user
FOREIGN KEY (user_id) REFERENCES users(id)
ON DELETE CASCADE;

-- Xóa foreign key
ALTER TABLE orders DROP FOREIGN KEY fk_orders_user;  -- MySQL
ALTER TABLE orders DROP CONSTRAINT fk_orders_user;  -- PostgreSQL
```

Sau khi thay đổi schema:

## Thực Hành Tốt

### Thiết Kế Schema

1. **Kiểu dữ liệu phù hợp**: Chọn loại nhỏ nhất đủ dùng
2. **Index cho query**: Index cột dùng trong WHERE và JOIN
3. **Định nghĩa foreign key**: Duy trì toàn vẹn tham chiếu
4. **Dùng NOT NULL**: Khi cột phải luôn có giá trị
5. **Thêm mặc định**: Giá trị mặc định hợp lý cho cột tùy chọn

### Quy Ước Đặt Tên

| Phần tử | Quy ước | Ví dụ |
|---------|------------|---------|
| Bảng | Số nhiều, snake_case | `users`, `order_items` |
| Cột | Số ít, snake_case | `user_id`, `created_at` |
| Index | `idx_table_columns` | `idx_users_email` |
| Foreign Key | `fk_table_reference` | `fk_orders_user` |
| Primary Key | `id` hoặc `table_id` | `id`, `user_id` |

### Cân Nhắc Hiệu Suất

1. **Primary key**: Mọi bảng nên có
2. **Index foreign key**: Index cột foreign key
3. **Index tổng hợp**: Sắp xếp cột theo tính chọn lọc
4. **Tránh over-indexing**: Index làm chậm ghi

## MongoDB Collections

MongoDB có mô hình cấu trúc khác SQL.

### Xem Schema

Tab cấu trúc MongoDB là **chỉ đọc**. MongoDB không có schema cố định, TablePro suy luận bằng cách lấy mẫu tài liệu. Hiển thị ba cột:

- **Name**: Tên trường (bao gồm đường dẫn lồng nhau dấu chấm)
- **Type**: Kiểu BSON (ObjectId, String, Int32, Int64, Double, Boolean, Date, Array, Object, v.v.)
- **Nullable**: Trường có trong tất cả tài liệu mẫu hay không

### Indexes

Tab Indexes hiển thị index collection. Dạng lệnh `createIndex()` có thể copy và chạy trong `mongosh`.

### Tab DDL

Tab DDL MongoDB hiển thị:
- **Định nghĩa index** dạng `db.collection.createIndex()`
- **Validator** dạng `db.runCommand({collMod: ...})`
- **Tùy chọn collection** (capped, size, max) nếu có

<Note>
Sửa đổi cấu trúc (thêm/xóa trường) không khả dụng cho MongoDB vì không có schema cố định. Chỉnh sửa tài liệu trực tiếp trong data grid.
</Note>

## Khắc Phục Sự Cố

### Cấu Trúc Không Tải

1. Kiểm tra kết nối database
2. Xác minh quyền xem schema
3. Refresh kết nối

### Thiếu Cột

1. Refresh cấu trúc bảng
2. Kiểm tra cột có trong transaction chưa commit không
3. Xác minh đúng database/schema

### Lỗi Cú Pháp DDL

Khi copy DDL sang database khác:

1. Kiểm tra khác biệt cú pháp theo database
2. Điều chỉnh kiểu dữ liệu cho database đích
3. Xóa tính năng không hỗ trợ (ví dụ: MySQL ENGINE cho PostgreSQL)
