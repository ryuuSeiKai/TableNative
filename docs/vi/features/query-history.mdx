# Lịch sử Truy vấn

Mọi query bạn chạy đều tự động lưu vào database SQLite cục bộ. Tìm kiếm bằng full-text search, lọc theo kết nối hoặc thời gian, và chạy lại query cũ. Lịch sử tồn tại qua các phiên làm việc.

## Truy Cập Lịch Sử

### Panel Lịch Sử

Mở panel lịch sử:

1. Click **View** > **Query History**
2. Hoặc nhấn `Cmd+Y`
3. Hoặc click biểu tượng **History** trong toolbar

### Lịch Sử Trong Sidebar

Query gần đây cũng hiện trong sidebar dưới kết nối đang hoạt động.

## Tính Năng Lịch Sử

### Danh Sách Query

Panel lịch sử hiển thị:

| Cột | Mô tả |
|--------|-------------|
| **Query** | Văn bản SQL (cắt ngắn nếu dài) |
| **Time** | Thời điểm chạy query |
| **Duration** | Thời gian thực thi |
| **Status** | Thành công hoặc lỗi |
| **Database** | Database/kết nối sử dụng |

### Sắp Xếp

Sắp xếp lịch sử theo:

- **Time**: Gần nhất trước (mặc định)
- **Duration**: Query chậm nhất trước
- **Query text**: Theo bảng chữ cái

Click header cột để đổi thứ tự.

### Lọc

Lọc query theo:

- **Connection**: Chỉ hiện query từ kết nối cụ thể
- **Status**: Chỉ hiện query thành công hoặc thất bại
- **Time range**: Hôm nay, tuần này, tháng này, tất cả

## Tìm Kiếm Lịch Sử

### Full-Text Search

Gõ vào ô tìm kiếm để tìm query chứa văn bản cụ thể:

```
SELECT users  -- Tìm tất cả query chứa "SELECT" và "users"
```

### Mẹo Tìm Kiếm

- Không phân biệt hoa thường
- Khớp một phần
- Tìm trên toàn bộ văn bản query

**Ví dụ**:

| Tìm kiếm | Tìm thấy |
|--------|-------|
| `users` | Query đề cập "users" |
| `JOIN orders` | Query join bảng orders |
| `WHERE created` | Query lọc theo created |
| `UPDATE users SET` | Câu lệnh update trên users |

## Sử Dụng Lịch Sử

### Chạy Lại Query

Chạy lại query từ lịch sử:

1. Tìm query trong danh sách
2. Double-click vào query
3. Query tải vào SQL editor
4. Nhấn `Cmd+Enter` để chạy

Hoặc click phải, chọn **Run Query**.

### Sao Chép Query

Sao chép query:

1. Chọn query trong danh sách
2. Click phải > **Copy Query**
3. Hoặc nhấn `Cmd+C`

Toàn bộ văn bản query sao chép vào clipboard.

### Chỉnh Sửa Trước Khi Chạy

Sửa query lịch sử:

1. Double-click để tải vào editor
2. Sửa đổi
3. Chạy query đã sửa

Entry gốc được giữ nguyên. Entry mới tạo cho query đã sửa.

## Lưu Trữ Lịch Sử

### Lịch Sử Lưu Ở Đâu

Lịch sử query nằm tại:

```
~/Library/Application Support/TablePro/query_history.db
```

Database SQLite này chứa:

- Văn bản query
- Timestamp thực thi
- Thời lượng
- Thông tin kết nối/database
- Trạng thái thành công/thất bại
- Thông báo lỗi (nếu có)

### Giữ Lại Lịch Sử

Cấu hình trong **Settings** > **History**:

| Thiết lập | Mặc định | Mô tả |
|---------|---------|-------------|
| **Max Entries** | 10,000 | Số query tối đa lưu |
| **Max Days** | 90 | Xóa query cũ hơn số ngày này |
| **Auto Cleanup** | Bật | Tự động xóa entry cũ |

<Tip>
Đặt **Max Entries** thành 0 cho lịch sử không giới hạn (có thể làm chậm tìm kiếm với lịch sử rất lớn).
</Tip>

### Xóa Lịch Sử

Xóa tất cả lịch sử:

1. Mở **Settings** > **History**
2. Click **Clear All History**
3. Xác nhận

<Warning>
Xóa lịch sử không thể hoàn tác. Cân nhắc export query quan trọng trước.
</Warning>

Xóa lịch sử cho kết nối cụ thể:

1. Click phải vào kết nối trong sidebar
2. Chọn **Clear History**

## Thực Hành Tốt Nhất

### Chú Thích Query

Thêm comment vào query quan trọng để tìm dễ hơn:

```sql
-- Báo cáo doanh thu tháng 2024
SELECT
    DATE_FORMAT(created_at, '%Y-%m') as month,
    SUM(total) as revenue
FROM orders
WHERE created_at >= '2024-01-01'
GROUP BY month;
```

Giờ tìm "Báo cáo doanh thu tháng" là ra.

### Tổ Chức Với Comment

Dùng mẫu comment nhất quán:

```sql
-- [REPORT] Tăng trưởng người dùng
-- [DEBUG] Vấn đề hiệu suất
-- [ADMIN] Dọn dẹp dữ liệu
-- [TEMP] Query thử nghiệm
```

### Export Query Quan Trọng

Query cần giữ lâu dài:

1. Tìm query trong lịch sử
2. Sao chép vào file SQL
3. Lưu trong version control

## Phím Tắt

| Hành động | Phím tắt |
|--------|----------|
| Mở lịch sử | `Cmd+Y` |
| Tìm kiếm lịch sử | `Cmd+F` (khi lịch sử đang mở) |
| Tải query đã chọn | `Enter` hoặc double-click |
| Sao chép query | `Cmd+C` |

## Khắc Phục Sự Cố

### Lịch Sử Không Lưu

1. Kiểm tra dung lượng đĩa
2. Xác minh TablePro có quyền ghi
3. Kiểm tra lịch sử có bị tắt trong cài đặt không

### Tìm Kiếm Chậm

Với lịch sử rất lớn:

1. Giảm **Max Entries** trong cài đặt
2. Xóa lịch sử cũ
3. Dùng từ khóa cụ thể hơn

### Thiếu Query Cũ

Kiểm tra **Max Days** -- query cũ hơn giới hạn này tự động bị xóa.

Để tắt xóa tự động:
1. Mở **Settings** > **History**
2. Đặt **Max Days** thành 0 (không giới hạn)

### Entry Trùng Lặp

Mỗi lần chạy tạo entry mới, dù query giống nhau. Đây là thiết kế có chủ ý: theo dõi từng lần query được chạy.
