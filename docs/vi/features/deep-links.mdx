TablePro hỗ trợ hai cách mở kết nối qua URL: database URL tiêu chuẩn (`mysql://`, `postgresql://`, v.v.) và URL scheme tùy chỉnh `tablepro://`.

## URL cơ sở dữ liệu

Mở kết nối database trực tiếp bằng URL tiêu chuẩn. Hoạt động từ terminal, script, Alfred, Raycast hoặc bất kỳ ứng dụng nào mở được URL.

### Các scheme được hỗ trợ

| Scheme | Database |
|--------|----------------|
| `mysql://` | MySQL / MariaDB |
| `postgresql://`, `postgres://` | PostgreSQL |
| `mongodb://`, `mongodb+srv://` | MongoDB |
| `redis://`, `rediss://` | Redis |
| `mssql://`, `sqlserver://` | SQL Server |
| `sqlite://` | SQLite (đường dẫn file cục bộ) |

### Định dạng URL

```
scheme://[username[:password]@]host[:port][/database][?param=value]
```

### Ví dụ

```bash
# MySQL
open "mysql://root@localhost/mydb" -a TablePro

# PostgreSQL với port và database
open "postgresql://admin@db.example.com:5432/analytics" -a TablePro

# MongoDB
open "mongodb://localhost:27017/mydb" -a TablePro

# Redis
open "redis://localhost:6379" -a TablePro

# SQLite (file cục bộ)
open "sqlite:///Users/me/data/app.db" -a TablePro
```

### Cách hoạt động

1. **Khớp kết nối đã lưu:** TablePro kiểm tra kết nối đã lưu nào khớp host, port, database và username của URL. Nếu tìm thấy, dùng lại kết nối đó (kể cả mật khẩu đã lưu).
2. **Khớp phiên đang mở:** Nếu kết nối đang hoạt động, TablePro đưa cửa sổ hiện có lên thay vì tạo bản sao.
3. **Kết nối tạm thời:** Không khớp kết nối nào, TablePro tạo kết nối tạm thời và kết nối ngay. Mật khẩu trong URL dùng cho lần kết nối đầu nhưng không lưu.

### Tham số truy vấn

Truyền thêm tùy chọn qua tham số:

| Tham số | Mô tả |
|---------|-------|
| `table` | Mở bảng cụ thể sau khi kết nối |
| `filter_column`, `filter_operation`, `filter_value` | Áp dụng filter trên bảng đã mở |
| `condition` | Áp dụng điều kiện WHERE SQL thô |
| `schema` | Chuyển sang schema (PostgreSQL) hoặc database (MySQL) cụ thể |

```bash
# Mở bảng với filter
open "mysql://root@localhost/mydb?table=users&filter_column=status&filter_operation=equal&filter_value=active" -a TablePro

# Mở với điều kiện SQL thô
open "postgresql://admin@localhost/mydb?table=orders&condition=total%20%3E%20100" -a TablePro

# Chuyển schema khi kết nối
open "postgresql://admin@localhost/mydb?schema=reporting" -a TablePro
```

<Warning>
Mật khẩu có thể đặt trong URL (`mysql://user:pass@host/db`), nhưng tránh chia sẻ URL chứa mật khẩu. Ưu tiên kết nối đã lưu với mật khẩu trong Keychain.
</Warning>

## Deep Links tùy chỉnh (tablepro://)

TablePro đăng ký URL scheme `tablepro://` để mở kết nối, bảng và query từ trình duyệt, terminal, tin nhắn Slack hoặc bất kỳ ứng dụng nào hỗ trợ link.

## Các mẫu URL

### Mở kết nối

```
tablepro://connect/{tên}
```

Mở kết nối đã lưu khớp `{tên}`. Không tìm thấy thì hiện thông báo lỗi.

```bash
open "tablepro://connect/Production"
```

### Mở bảng

```
tablepro://connect/{tên}/table/{bảng}
tablepro://connect/{tên}/database/{db}/table/{bảng}
```

Kết nối và mở bảng chỉ định. Dạng thứ hai nhắm đến database cụ thể trên kết nối.

```bash
open "tablepro://connect/Production/table/users"
open "tablepro://connect/Production/database/analytics/table/events"
```

### Mở query

```
tablepro://connect/{tên}/query?sql={sql_đã_mã_hóa}
```

Kết nối và mở tab query mới với SQL được điền sẵn. SQL phải percent-encoded.

```bash
open "tablepro://connect/Production/query?sql=SELECT%20*%20FROM%20users%20LIMIT%2010"
```

### Nhập kết nối

```
tablepro://import?name={n}&host={h}&port={p}&type={t}&username={u}&database={db}
```

Tạo kết nối mới và mở form kết nối để xem lại. Thêm mật khẩu trước khi kết nối.

```bash
open "tablepro://import?name=Staging&host=db.example.com&port=5432&type=postgresql&username=admin&database=mydb"
```

**Tham số bắt buộc:** `name`, `host`, `type`
**Tham số tùy chọn:** `port` (mặc định cổng tiêu chuẩn theo loại database), `username`, `database`

Giá trị `type` hỗ trợ: `MySQL`, `MariaDB`, `PostgreSQL`, `SQLite`, `MongoDB` (không phân biệt hoa thường).

## Chi tiết hoạt động

- **Khớp tên kết nối** không phân biệt hoa thường. `Production`, `production` và `PRODUCTION` đều khớp cùng kết nối.
- **Khoảng trắng và ký tự đặc biệt** trong tên được trình duyệt và macOS tự động percent-encode.
- **Đã kết nối:** nếu kết nối đang mở, TablePro bỏ qua kết nối lại và mở tab được yêu cầu trực tiếp.
- **Ứng dụng chưa chạy:** macOS khởi chạy TablePro trước, sau đó chuyển URL.

<Warning>
Mật khẩu không bao giờ có trong URL deep link. Kết nối được import yêu cầu nhập mật khẩu thủ công trong form kết nối.
</Warning>

## Sử dụng từ Terminal

Dùng lệnh `open` để kích hoạt deep link từ terminal:

```bash
# Kết nối
open "tablepro://connect/Production"

# Mở bảng cụ thể
open "tablepro://connect/Production/table/users"

# Chạy query
open "tablepro://connect/Production/query?sql=SELECT%20count(*)%20FROM%20orders"

# Nhập kết nối mới để xem lại
open "tablepro://import?name=Dev&host=localhost&port=3306&type=mysql&username=root&database=app"
```
