# Hướng dẫn Bắt đầu Nhanh

Tải, cài đặt và chạy query đầu tiên.

## Bước 1: Tải xuống TablePro

Cách nhanh nhất là dùng [Homebrew](https://brew.sh):

```bash
brew install --cask tablepro
```

<Tip>
Nếu cài qua Homebrew, chuyển thẳng đến [Bước 3](#bước-3-tạo-kết-nối-đầu-tiên). Công thức cask xử lý cài đặt và tự động xóa thuộc tính quarantine của macOS.
</Tip>

Hoặc tải thủ công:

1. Truy cập [trang GitHub Releases](https://github.com/datlechin/tablepro/releases)
2. Tải file DMG cho Mac:
   - **Apple Silicon** (M1/M2/M3/M4/M5): `TablePro-arm64.dmg`
   - **Intel**: `TablePro-x86_64.dmg`

<Note>
Không chắc Mac dùng chip nào? Nhấp menu Apple, chọn **About This Mac**. Tìm "Chip" (Apple Silicon) hoặc "Processor" (Intel).
</Note>

## Bước 2: Cài đặt TablePro

1. Mở file DMG đã tải
2. Kéo **TablePro** vào thư mục **Applications**
3. Eject DMG
4. Mở TablePro từ thư mục Applications

<Warning>
Lần mở đầu tiên, macOS sẽ báo lỗi **"TablePro is damaged and can't be opened"** vì app chưa được ký chứng chỉ Apple Developer. Mở **Terminal** và chạy:

```bash
xattr -cr /Applications/TablePro.app
```

Sau đó mở lại TablePro. Chỉ cần làm một lần. Xem [hướng dẫn Cài đặt](/vi/installation#bảo-mật-khởi-chạy-lần-đầu) để biết thêm.
</Warning>

## Bước 3: Tạo Kết nối Đầu tiên

Màn hình Welcome hiện ra khi khởi chạy. Tạo kết nối database mới từ đây.

### Đối với MySQL/MariaDB

1. Nhấp **New Connection**
2. Chọn **MySQL** hoặc **MariaDB**
3. Điền thông tin kết nối:
   - **Name**: Tên gợi nhớ (ví dụ: "Local MySQL")
   - **Host**: `localhost` (hoặc địa chỉ server)
   - **Port**: `3306` (mặc định)
   - **Username**: Tên user database (ví dụ: `root`)
   - **Password**: Mật khẩu database
   - **Database**: (Tùy chọn) Database mặc định

### Đối với PostgreSQL

1. Nhấp **New Connection**
2. Chọn **PostgreSQL**
3. Điền thông tin kết nối:
   - **Name**: Tên gợi nhớ (ví dụ: "Local PostgreSQL")
   - **Host**: `localhost` (hoặc địa chỉ server)
   - **Port**: `5432` (mặc định)
   - **Username**: Tên user database (ví dụ: `postgres`)
   - **Password**: Mật khẩu database
   - **Database**: Tên database

### Đối với Amazon Redshift

1. Nhấp **New Connection**
2. Chọn **Amazon Redshift**
3. Điền thông tin kết nối:
   - **Name**: Tên gợi nhớ (ví dụ: "Redshift Warehouse")
   - **Host**: Endpoint cluster (ví dụ: `my-cluster.abc123.us-east-1.redshift.amazonaws.com`)
   - **Port**: `5439` (mặc định)
   - **Username**: Tên user Redshift (ví dụ: `awsuser`)
   - **Password**: Mật khẩu
   - **Database**: Tên database (thường là `dev`)

### Đối với SQLite

1. Nhấp **New Connection**
2. Chọn **SQLite**
3. Nhấp **Browse** để chọn file database SQLite
4. Đặt tên cho kết nối

<Tip>
SQLite không cần xác thực. Chỉ cần chọn file `.sqlite` hoặc `.db` trên Mac.
</Tip>

### Đối với MongoDB

1. Nhấp **New Connection**
2. Chọn **MongoDB**
3. Điền thông tin kết nối:
   - **Name**: Tên gợi nhớ (ví dụ: "Local MongoDB")
   - **Host**: `localhost` (hoặc địa chỉ server)
   - **Port**: `27017` (mặc định)
   - **Username**: Tên user database (nếu bật xác thực)
   - **Password**: Mật khẩu database
   - **Database**: Tên database
   - **Auth Database**: Database xác thực (mặc định là `admin`)

<Tip>
MongoDB cục bộ không có xác thực chỉ cần host và port. Kết nối sử dụng giao thức `mongodb://` bên trong.
</Tip>

## Bước 4: Kiểm tra và Kết nối

1. Nhấp **Test Connection** để kiểm tra cài đặt
2. Dấu kiểm xanh xác nhận kết nối thành công
3. Nhấp **Create** để lưu và kết nối

## Bước 5: Khám phá Cơ sở dữ liệu

Giao diện chính gồm ba phần:

- **Sidebar**: Database và bảng
- **SQL Editor**: Viết và thực thi truy vấn SQL
- **Data Grid**: Xem và chỉnh sửa kết quả truy vấn

### Thử Truy vấn Đầu tiên

1. Nhấp vào một bảng trong sidebar để xem nội dung
2. Hoặc gõ query trong SQL editor:

```sql
SELECT * FROM users LIMIT 10;
```

3. Nhấn `Cmd+Enter` để thực thi
4. Xem kết quả trong data grid bên dưới
