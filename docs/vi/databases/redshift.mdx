# Kết nối Amazon Redshift

TablePro kết nối Amazon Redshift, kho dữ liệu dạng cột của AWS dựa trên PostgreSQL 8.0.2. Kết nối dùng cùng driver libpq như PostgreSQL, với truy vấn metadata riêng cho distribution style, sort keys và kích thước bảng.

## Thiết lập nhanh

<Steps>
  <Step title="Mở Form kết nối">
    Nhấp **New Connection** từ màn hình Chào mừng hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn Redshift">
    Chọn **Amazon Redshift** từ danh sách loại database
  </Step>
  <Step title="Nhập thông tin kết nối">
    Điền cluster endpoint, port, username, password và tên database
  </Step>
  <Step title="Kiểm tra và kết nối">
    Nhấp **Test Connection**, rồi **Create**
  </Step>
</Steps>

## Cài đặt kết nối

### Trường bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Tên định danh kết nối | - |
| **Host** | Cluster endpoint Redshift | - |
| **Port** | Cổng Redshift | `5439` |
| **Username** | Người dùng Redshift | - |
| **Password** | Mật khẩu | - |
| **Database** | Tên database | `dev` |

<Warning>
Redshift bắt buộc tên database. Database mặc định mỗi cluster là `dev`.
</Warning>

## Ví dụ cấu hình

### Redshift Cluster

```
Name:     Production Warehouse
Host:     my-cluster.abc123xyz.us-east-1.redshift.amazonaws.com
Port:     5439
Username: awsuser
Password: (mật khẩu)
Database: dev
```

### Redshift Serverless

```
Name:     Serverless Warehouse
Host:     my-workgroup.123456789012.us-west-2.redshift-serverless.amazonaws.com
Port:     5439
Username: admin
Password: (mật khẩu)
Database: dev
```

<Tip>
Tìm cluster endpoint trong AWS Console tại Amazon Redshift > Clusters > cluster > Properties. Endpoint gồm port, nhưng nhập host và port riêng trong TablePro.
</Tip>

## Tính năng

### Hỗ trợ Schema

Redshift tổ chức bảng thành schemas giống PostgreSQL. TablePro hiển thị:

1. Tất cả schemas user có quyền truy cập
2. Các bảng trong mỗi schema
3. Thứ tự search path

### Chuyển đổi Schema

Chuyển schema Redshift bằng switcher (**Cmd+K**):

1. Nhấn **Cmd+K** để mở switcher
2. Chọn schema đích
3. Sidebar, truy vấn và toolbar cập nhật theo schema đã chọn

<Tip>
Schema mặc định là `public`. Schema hệ thống (`pg_catalog`, `information_schema`) được lọc khỏi danh sách.
</Tip>

### Duyệt bảng và xem dữ liệu

Duyệt bảng, views và late-binding views. Data grid phân trang cho bảng warehouse lớn. Metadata Redshift từ `svv_table_info` có trong panel thông tin bảng: distribution style, sort keys và kích thước.

### Thực thi truy vấn

Hỗ trợ cú pháp Redshift SQL. Editor highlight các hàm và từ khóa riêng Redshift.

```sql
-- Window functions
SELECT
    category,
    product,
    revenue,
    RANK() OVER (PARTITION BY category ORDER BY revenue DESC) as rank
FROM sales;

-- COPY từ S3
COPY sales FROM 's3://mybucket/data/sales/'
IAM_ROLE 'arn:aws:iam::123456789012:role/MyRedshiftRole'
FORMAT AS PARQUET;

-- UNLOAD ra S3
UNLOAD ('SELECT * FROM sales WHERE year = 2024')
TO 's3://mybucket/export/sales_2024_'
IAM_ROLE 'arn:aws:iam::123456789012:role/MyRedshiftRole'
FORMAT AS CSV;

-- Phân tích compression
ANALYZE COMPRESSION sales;
```

### Xuất dữ liệu

Xuất kết quả truy vấn hoặc dữ liệu bảng nhiều định dạng:

- CSV
- JSON
- SQL (câu lệnh INSERT)
- XLSX

### Nhập dữ liệu

Import từ CSV, JSON, SQL và XLSX vào bảng Redshift.

### DDL

Xem DDL (CREATE TABLE) cho bất kỳ bảng nào, gồm mệnh đề Redshift: DISTKEY, SORTKEY, DISTSTYLE và ENCODE.

### Điều hướng Foreign Key

Redshift hỗ trợ định nghĩa foreign key, nhưng chỉ mang tính thông tin (engine không thực thi). TablePro hiển thị quan hệ này và cho phép điều hướng giữa bảng liên quan.

## SSL/TLS

Kết nối Redshift hỗ trợ SSL/TLS, cấu hình giống PostgreSQL:

| Chế độ SSL | Mô tả |
|----------|-------------|
| **Disabled** | Không mã hóa SSL |
| **Preferred** | Dùng SSL nếu có, không thì kết nối thường |
| **Required** | Bắt buộc SSL, không xác minh chứng chỉ |
| **Verify CA** | Bắt buộc SSL và xác minh chứng chỉ server với CA |
| **Verify Identity** | Bắt buộc SSL, xác minh CA và hostname server |

<Tip>
AWS Redshift hỗ trợ SSL mặc định. Dùng **Required** hoặc **Verify CA** cho kết nối production.
</Tip>

## Hỗ trợ SSH Tunnel

Kết nối Redshift qua SSH tunnel nếu cluster nằm trong VPC private. Cấu hình trong tab SSH của form kết nối. TablePro định tuyến kết nối qua tunnel.

## Hạn chế

Redshift là kho dữ liệu dạng cột, không phải RDBMS đa năng. Một số tính năng PostgreSQL không có hoặc khác:

| Tính năng | Hành vi Redshift |
|---------|------------------|
| **Indexes** | Không có B-tree. Dùng DISTKEY và SORTKEY. Hiển thị trong tab Structure. |
| **ALTER TABLE** | Hạn chế. Không đổi tên cột. Thay đổi kiểu chỉ mở rộng VARCHAR hoặc đổi độ chính xác số. |
| **Enum types** | Không hỗ trợ. |
| **Sequences** | Không hỗ trợ. Dùng IDENTITY. |
| **Foreign keys** | Chỉ thông tin. Định nghĩa DDL nhưng engine không thực thi. |
| **Triggers** | Không hỗ trợ. |
| **Stored procedures** | Hỗ trợ, nhưng PL/pgSQL giới hạn. |

### DISTKEY và SORTKEY

Thay vì index truyền thống, Redshift phân phối dữ liệu qua node bằng distribution keys và sắp xếp bằng sort keys. TablePro hiển thị DISTKEY và SORTKEY trong tab Structure, cho thấy cách dữ liệu tổ chức vật lý.

## Khắc phục sự cố

### Connection Refused

**Triệu chứng**: "Connection refused" hoặc timeout

**Nguyên nhân**:

1. **Security group chưa cấu hình**: IP phải được phép trong VPC security group của cluster (inbound rule port 5439).
2. **Cluster không public**: Nếu `Publicly Accessible` là No, cần SSH tunnel hoặc VPN.
3. **Cluster tạm dừng**: Redshift cluster có thể pause để tiết kiệm. Resume trong AWS Console.

### Authentication Failed

**Triệu chứng**: "password authentication failed"

**Giải pháp**:

1. Kiểm tra master username và password trong AWS Console
2. Cho IAM auth, đảm bảo credentials còn hiệu lực
3. Kiểm tra user được cấp quyền truy cập database

### Hiệu năng truy vấn

Redshift tối ưu cho truy vấn phân tích trên dataset lớn. Nếu chậm:

1. Kiểm tra distribution style: `SELECT * FROM svv_table_info WHERE "table" = 'your_table'`
2. Chạy `ANALYZE` cập nhật thống kê
3. Dùng `EXPLAIN` xem query plan
4. Đảm bảo SORTKEY khớp predicate WHERE và JOIN thường dùng

## Khác biệt với PostgreSQL

Nếu bạn từ PostgreSQL:

| PostgreSQL | Redshift |
|------------|----------|
| Cổng mặc định 5432 | Cổng mặc định 5439 |
| Index B-tree, GIN, GiST | Chỉ DISTKEY + SORTKEY |
| `CREATE INDEX` | Không hỗ trợ |
| Constraint đầy đủ | Foreign key chỉ thông tin |
| `SERIAL` / `GENERATED AS IDENTITY` | `IDENTITY(seed, step)` |
| Kiểu phong phú (JSONB, arrays, v.v.) | Kiểu hạn chế (không arrays, JSON qua SUPER) |
| `pg_stat_activity` | `stv_recents`, `svl_qlog` |
| Lưu trữ theo dòng | Lưu trữ theo cột |
