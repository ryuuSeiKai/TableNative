# Kết nối Microsoft SQL Server

TablePro kết nối SQL Server 2017 trở lên qua FreeTDS. Hỗ trợ instance trên Windows, Linux, Docker và Azure SQL Database.

<Warning>
FreeTDS phải được build trước khi kết nối SQL Server. Chạy `scripts/build-freetds.sh` một lần để biên dịch thư viện.
</Warning>

## Thiết lập nhanh

<Steps>
  <Step title="Build FreeTDS">
    Chạy `scripts/build-freetds.sh` từ thư mục gốc repository
  </Step>
  <Step title="Mở Form kết nối">
    Nhấp **New Connection** từ màn hình Chào mừng hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn SQL Server">
    Chọn **SQL Server** từ danh sách loại database
  </Step>
  <Step title="Nhập thông tin kết nối">
    Điền host, port, username, password và tên database
  </Step>
  <Step title="Kiểm tra và kết nối">
    Nhấp **Test Connection**, rồi **Create**
  </Step>
</Steps>

## Cài đặt kết nối

### Trường bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Tên định danh kết nối | - |
| **Host** | Hostname hoặc IP server | `localhost` |
| **Port** | Cổng SQL Server | `1433` |
| **Username** | Tên đăng nhập SQL Server | `sa` |
| **Password** | Mật khẩu | - |
| **Database** | Tên database | - |

<Note>
TablePro dùng SQL Server Authentication (username và password). Windows Authentication không hỗ trợ.
</Note>

## Ví dụ cấu hình

### Server dev local

```
Name:     Local SQL Server
Host:     localhost
Port:     1433
Username: sa
Password: (mật khẩu)
Database: master
```

### Docker SQL Server Container

Khởi động SQL Server 2022 container:

```bash
docker run \
  -e "ACCEPT_EULA=Y" \
  -e "SA_PASSWORD=YourStrong@Passw0rd" \
  -p 1433:1433 \
  --name sql-server \
  -d mcr.microsoft.com/mssql/server:2022-latest
```

Kết nối:

```
Name:     Docker SQL Server
Host:     localhost
Port:     1433
Username: sa
Password: YourStrong@Passw0rd
Database: master
```

<Tip>
`SA_PASSWORD` phải đáp ứng yêu cầu phức tạp SQL Server: tối thiểu 8 ký tự gồm chữ hoa, chữ thường, số và ký hiệu.
</Tip>

### Server từ xa

```
Name:     Production SQL Server
Host:     sql.example.com
Port:     1433
Username: app_user
Password: (mật khẩu an toàn)
Database: myapp_production
```

### Azure SQL Database

```
Name:     Azure SQL
Host:     myserver.database.windows.net
Port:     1433
Username: admin@myserver
Password: (mật khẩu)
Database: mydb
```

<Note>
Azure SQL Database có thể yêu cầu username gồm hậu tố server (ví dụ: `user@servername`) tùy cấu hình xác thực.
</Note>

## Tính năng

### Chọn Schema

SQL Server tổ chức đối tượng thành schemas trong database. Schema mặc định là `dbo`. TablePro hiển thị tất cả schemas login có quyền truy cập và liệt kê bảng trong mỗi schema.

Chuyển schema bằng switcher (**Cmd+K**), hiển thị schemas cho database hiện tại.

### Duyệt bảng

Mỗi bảng hiển thị:

- **Structure**: Cột với kiểu SQL Server, nullable và giá trị mặc định
- **Indexes**: Clustered và non-clustered indexes
- **Foreign Keys**: Quan hệ với bảng khác
- **DDL**: Định nghĩa bảng gồm constraints

### Query Editor

Viết và chạy T-SQL trong SQL editor. Phân trang dùng cú pháp offset chuẩn SQL Server:

```sql
-- Kết quả phân trang
SELECT *
FROM dbo.orders
ORDER BY order_id
OFFSET 0 ROWS FETCH NEXT 50 ROWS ONLY;

-- Ước tính số dòng (nhanh, dùng thống kê)
SELECT SUM(p.rows) AS row_count
FROM sys.partitions p
JOIN sys.tables t ON p.object_id = t.object_id
WHERE p.index_id IN (0, 1)
  AND t.name = 'orders';

-- Định nghĩa view
SELECT OBJECT_DEFINITION(OBJECT_ID('dbo.my_view'));

-- Liệt kê bảng
SELECT TABLE_SCHEMA, TABLE_NAME
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_TYPE = 'BASE TABLE'
ORDER BY TABLE_SCHEMA, TABLE_NAME;
```

### Chỉnh sửa Schema

Sửa cấu trúc bảng bằng T-SQL ALTER TABLE. SQL Server dùng bracket cho identifiers:

```sql
-- Thêm cột
ALTER TABLE [dbo].[users] ADD [last_login] DATETIME2 NULL;

-- Đổi tên cột (dùng sp_rename)
EXEC sp_rename 'dbo.users.old_name', 'new_name', 'COLUMN';

-- Xóa cột
ALTER TABLE [dbo].[users] DROP COLUMN [legacy_field];

-- Thêm index
CREATE INDEX [IX_users_email] ON [dbo].[users] ([email]);
```

### Foreign Keys

TablePro hiển thị foreign key trong view cấu trúc bảng. Xem thủ công:

```sql
SELECT
    fk.name AS fk_name,
    OBJECT_NAME(fk.parent_object_id) AS parent_table,
    COL_NAME(fkc.parent_object_id, fkc.parent_column_id) AS parent_column,
    OBJECT_NAME(fk.referenced_object_id) AS referenced_table,
    COL_NAME(fkc.referenced_object_id, fkc.referenced_column_id) AS referenced_column
FROM sys.foreign_keys fk
JOIN sys.foreign_key_columns fkc ON fk.object_id = fkc.constraint_object_id
ORDER BY parent_table, fk_name;
```

## Xác thực

TablePro chỉ dùng SQL Server Authentication. Nhập login và password trong form kết nối.

**Windows Authentication** (Integrated Security / Kerberos) không hỗ trợ.

Tạo login SQL Server:

```sql
-- Tạo login
CREATE LOGIN app_user WITH PASSWORD = 'StrongPassword1!';

-- Tạo database user
USE mydb;
CREATE USER app_user FOR LOGIN app_user;

-- Cấp quyền
ALTER ROLE db_datareader ADD MEMBER app_user;
ALTER ROLE db_datawriter ADD MEMBER app_user;
```

## Khắc phục sự cố

### Connection Refused

**Triệu chứng**: "Unable to connect" hoặc timeout

**Nguyên nhân và Giải pháp**:

1. **SQL Server không chạy hoặc không chấp nhận TCP**

   SQL Server Browser và TCP/IP phải bật:
   - Mở SQL Server Configuration Manager
   - Bật TCP/IP trong "SQL Server Network Configuration"
   - Đảm bảo dịch vụ SQL Server đang chạy

2. **Cổng 1433 bị firewall chặn**
   - Windows Firewall: thêm inbound rule TCP port 1433
   - Cloud VM: kiểm tra security group / firewall rules

3. **Docker container chưa khởi động**
   ```bash
   docker start sql-server
   docker logs sql-server
   ```

### Đăng nhập thất bại

**Triệu chứng**: "Login failed for user"

**Giải pháp**:

1. Kiểm tra username và password
2. Xác nhận SQL Server Authentication bật (không chỉ Windows-only):
   ```sql
   -- Kiểm tra chế độ xác thực (1 = chỉ Windows, 2 = mixed mode)
   SELECT SERVERPROPERTY('IsIntegratedSecurityOnly');
   ```
3. Bật mixed-mode trong SQL Server Management Studio tại Server Properties > Security, rồi restart SQL Server

### FreeTDS không tìm thấy

**Triệu chứng**: "FreeTDS library not found" hoặc lỗi linker

**Giải pháp**: Chạy build script:

```bash
scripts/build-freetds.sh
```

Script biên dịch FreeTDS và đặt static library đúng vị trí Xcode project yêu cầu.

## Hạn chế

- **Windows Authentication không hỗ trợ.** Chỉ SQL Server Authentication (username + password).
- **Named instances** (ví dụ: `HOSTNAME\INSTANCENAME`) chưa hỗ trợ. Kết nối qua IP và port rõ ràng.
- **Cột `NTEXT`, `TEXT`, `IMAGE`** (deprecated từ SQL Server 2005) có thể hạn chế chỉnh sửa. Dùng `NVARCHAR(MAX)`, `VARCHAR(MAX)`, `VARBINARY(MAX)` cho schema mới.
