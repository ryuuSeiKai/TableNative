## Tạo Kết nối

### Từ Màn hình Chào mừng

Màn hình Chào mừng hiện ra khi khởi chạy lần đầu hoặc khi không có kết nối nào đang hoạt động.

1. Nhấp **New Connection**
2. Điền thông tin kết nối
3. Nhấp **Test Connection** để kiểm tra
4. Nhấp **Create** để lưu và kết nối

### Từ Menu Bar

Tạo kết nối mới bất cứ lúc nào:

- **File** > **New Connection** (`Cmd+N`)

### Từ URL Kết nối

Các nền tảng hosting, công cụ triển khai và tài liệu kỹ thuật thường cung cấp chuỗi kết nối. Dán trực tiếp vào form kết nối:

1. Mở form **New Connection**
2. Nhấp **Import from URL** trong tab General
3. Dán URL kết nối vào hộp thoại và nhấp **Import**
4. Tất cả các trường (loại, host, port, database, username, password) được điền tự động
5. Kiểm tra lại các giá trị rồi nhấp **Create**

Xem [Tham chiếu URL kết nối](/databases/connection-urls) để biết tất cả scheme và định dạng được hỗ trợ.

<Tip>
URL có ký tự đặc biệt trong mật khẩu (như `@`, `#` hoặc `%`) cần dùng percent-encoding. Ví dụ: `p@ssword` thành `p%40ssword`.
</Tip>

### Kết nối Trực tiếp từ URL

Mở URL cơ sở dữ liệu từ trình duyệt hoặc terminal. TablePro khởi động và kết nối ngay, không cần form hay kết nối đã lưu.

**Từ trình duyệt:** dán URL vào thanh địa chỉ và nhấn Enter.

**Từ terminal:**

```bash
open "postgresql://user:pass@host:5432/dbname"
open "mysql://root:secret@localhost:3306/myapp"
open "redis://:password@localhost:6379"
```

TablePro đăng ký `postgresql`, `postgres`, `mysql`, `mariadb`, `sqlite`, `mongodb`, `mongodb+srv`, `redis`, `rediss`, `redshift`, `mssql`, `sqlserver`, `oracle`, `clickhouse`, `duckdb`, `cassandra` và `scylladb` là các URL scheme trên macOS, nên hệ điều hành sẽ chuyển hướng các URL này trực tiếp đến ứng dụng.

**Khi mở URL:**

- Nếu có kết nối đã lưu khớp host, port, database và username, TablePro sẽ dùng lại kết nối đó
- Nếu không khớp, một phiên tạm thời được tạo. Không có gì được lưu vào danh sách kết nối
- Mật khẩu từ URL được lưu trong Keychain trong suốt phiên làm việc

Bạn cũng có thể chỉ định bảng, schema cụ thể hoặc áp dụng bộ lọc qua query parameter.

Xem [Tham chiếu URL kết nối](/databases/connection-urls) để biết tất cả tham số truy vấn được hỗ trợ.

<Note>
Khác với **Import from URL** (điền form để bạn xem xét và lưu), mở URL trực tiếp bỏ qua form hoàn toàn và kết nối ngay lập tức.
</Note>

### Các trường Form Kết nối

#### Phần General

| Trường | Mô tả |
|-------|-------------|
| **Name** | Tên thân thiện để xác định kết nối này |
| **Type** | Loại cơ sở dữ liệu: MySQL, MariaDB, PostgreSQL, SQLite, MongoDB, Redis, Redshift, SQL Server, Oracle, ClickHouse, DuckDB, Cassandra hoặc ScyllaDB |

#### Phần Appearance

| Trường | Mô tả |
|-------|-------------|
| **Color** | Mã màu cho kết nối, dễ nhận biết nhanh |
| **Tag** | Gắn tag tùy chỉnh để phân loại |
| **Group** | Đưa kết nối vào nhóm (thư mục) |
| **Read-Only** | Bật để chặn mọi thao tác ghi trên kết nối này |

<Tip>
Dùng màu để phân biệt môi trường: đỏ cho production, xanh cho development. Bật **Read-Only** cho database production để tránh ghi nhầm.
</Tip>

#### Phần Connection

| Trường | Mô tả |
|-------|-------------|
| **Host** | Địa chỉ server (mặc định: `localhost`) |
| **Port** | Cổng server (tự động điền theo loại database) |
| **Database** | Database mặc định (tùy chọn cho MySQL/MariaDB) |

Với SQLite, phần này hiển thị trình duyệt file:

| Trường | Mô tả |
|-------|-------------|
| **File Path** | Đường dẫn đến file database SQLite |

#### Phần Authentication

Cho MySQL, MariaDB và PostgreSQL:

| Trường | Mô tả |
|-------|-------------|
| **Username** | Tên người dùng database (mặc định: `root` cho MySQL) |
| **Password** | Mật khẩu (lưu an toàn trong Keychain) |

<Note>
SQLite không cần xác thực.
</Note>

#### Phần SSL/TLS

Cho kết nối MySQL, MariaDB và PostgreSQL:

| Trường | Mô tả |
|-------|-------------|
| **SSL Mode** | Chế độ mã hóa (xem bảng bên dưới) |
| **CA Certificate** | Đường dẫn file Certificate Authority (cho Verify CA/Verify Identity) |
| **Client Certificate** | Đường dẫn file chứng chỉ client (tùy chọn) |
| **Client Key** | Đường dẫn file khóa riêng client (tùy chọn) |

**Các chế độ SSL:**

| Chế độ | Mô tả |
|------|-------------|
| **Disabled** | Không mã hóa SSL (mặc định) |
| **Preferred** | Dùng SSL nếu có, không thì kết nối thường |
| **Required** | Bắt buộc SSL, không xác minh chứng chỉ |
| **Verify CA** | Bắt buộc SSL và xác minh chứng chỉ server với CA |
| **Verify Identity** | Bắt buộc SSL, xác minh CA và hostname server khớp chứng chỉ |

<Tip>
Cho database production, dùng **Verify CA** hoặc **Verify Identity** để bảo mật tối đa. Cho server dev, **Required** mã hóa mà không cần file chứng chỉ.
</Tip>

<Note>
SSL/TLS không áp dụng cho SQLite (dạng file, không qua mạng).
</Note>

Khi kết nối có màu, thanh công cụ cửa sổ phủ nhẹ tông màu đó. Giúp bạn nhận biết ngay đang ở môi trường nào, tương tự cách Safari tô màu toolbar theo website.

Tông màu đủ nhẹ để không ảnh hưởng khả năng đọc nhưng vẫn dễ nhận ra.

## Nhóm Kết nối

Nhóm sắp xếp kết nối thành thư mục thu gọn được trên màn hình Chào mừng:

1. Chuột phải trong danh sách kết nối > **New Group**, hoặc nhấp biểu tượng thư mục cạnh thanh tìm kiếm
2. Đặt tên nhóm và chọn màu
3. Kéo kết nối vào nhóm, hoặc chỉ định nhóm trong phần **Appearance** của form

### Quản lý Nhóm

| Thao tác | Cách thực hiện |
|--------|-----|
| **Tạo** | Nhấp biểu tượng thư mục hoặc chuột phải > New Group |
| **Đổi tên** | Chuột phải tiêu đề nhóm > Rename |
| **Đổi màu** | Chuột phải tiêu đề nhóm > Change Color |
| **Xóa** | Chuột phải tiêu đề nhóm > Delete Group (kết nối chuyển về Ungrouped) |

Thu gọn/mở rộng nhóm bằng cách nhấp mũi tên. Trạng thái thu gọn được giữ giữa các phiên.

<Tip>
Kết hợp nhóm với màu sắc: đặt kết nối production vào nhóm "Production" tiêu đề đỏ, development vào nhóm "Development" xanh lá.
</Tip>

## Chuyển đổi Kết nối Nhanh

Chuyển đổi giữa các kết nối đang hoạt động hoặc đã lưu từ toolbar, không cần quay lại màn hình Chào mừng:

1. Nhấp nút **tên kết nối** trên toolbar
2. Popover hiển thị các phiên hoạt động và kết nối đã lưu
3. Nhấp kết nối để chuyển ngay
4. Nhấp **Manage Connections...** để mở trình quản lý đầy đủ

<Tip>
Đây là cách nhanh nhất để nhảy giữa các database trong lúc phát triển. Nhấp lại nút kết nối sẽ đóng popover.
</Tip>

## Kết nối Nhanh từ Dock Menu

Mở kết nối đã lưu trực tiếp từ Dock macOS, không cần qua màn hình Chào mừng.

Chuột phải biểu tượng TablePro trên Dock:

| Mục menu | Mô tả |
|-----------|-------------|
| **Show Welcome Window** | Mở màn hình Chào mừng |
| **Open Connection** | Menu con liệt kê tất cả kết nối đã lưu kèm icon loại database |

### Cách dùng

1. Chuột phải biểu tượng TablePro trên Dock
2. Di chuột qua **Open Connection**
3. Nhấp tên kết nối
4. TablePro mở và kết nối tự động

Nếu kết nối thất bại (ví dụ: server không truy cập được), TablePro chuyển về màn hình Chào mừng để bạn kiểm tra lại.

<Tip>
Mỗi kết nối trong Dock menu hiển thị icon loại database (MySQL, MariaDB, PostgreSQL, SQLite), dễ nhận biết đúng kết nối.
</Tip>

## Giám sát Sức khỏe Kết nối

TablePro giám sát kết nối đang hoạt động và tự phục hồi khi gián đoạn.

### Kiểm tra Tự động

Với MySQL, MariaDB và PostgreSQL, TablePro ping (`SELECT 1`) mỗi **30 giây** để xác minh kết nối còn sống. SQLite dạng file nên không cần giám sát.

### Tự động Kết nối lại

Khi phát hiện kết nối mất, TablePro kết nối lại với thời gian chờ tăng dần:

1. **Lần 1** -- chờ 2 giây, kết nối lại
2. **Lần 2** -- chờ 4 giây, kết nối lại
3. **Lần 3** -- chờ 8 giây, kết nối lại

Nếu cả 3 lần thất bại, kết nối chuyển sang trạng thái lỗi. Nút **Reconnect** xuất hiện trên toolbar.

SSH tunnel có giám sát sức khỏe riêng. Tunnel được thiết lập lại tự động nếu tiến trình bị dừng.

### Kết nối lại Thủ công

Nếu tự động kết nối lại thất bại, nhấp nút **Reconnect** trên toolbar. Với kết nối SSH, thao tác này cũng tạo lại tunnel.

<Tip>
Chỉ báo trạng thái trên toolbar: xanh lá = đã kết nối, cam = đang kết nối lại, đỏ = lỗi/đã ngắt.
</Tip>

<Note>
SQLite dạng file, không cần giám sát sức khỏe hay tự động kết nối lại.
</Note>

## Lệnh Khởi động

Lệnh khởi động là các câu lệnh SQL tự động chạy mỗi khi kết nối được thiết lập. Dùng để đặt biến phiên, cấu hình timezone, thay đổi encoding hoặc thiết lập các tùy chọn cấp phiên khác.

Cấu hình lệnh khởi động trong tab **Advanced** của form kết nối. Nhập mỗi câu lệnh SQL trên một dòng.

### Ví dụ Phổ biến

```sql
SET time_zone = '+00:00';
SET NAMES utf8mb4;
SET sql_mode = 'STRICT_TRANS_TABLES';
SET search_path TO myschema, public;
SET statement_timeout = '30s';
```

Lệnh khởi động thực thi theo thứ tự từ trên xuống dưới. Nếu một lệnh thất bại, kết nối vẫn tiếp tục nhưng lệnh lỗi sẽ bị bỏ qua.

<Tip>
Lệnh khởi động hữu ích để đảm bảo cài đặt phiên thống nhất giữa các thành viên trong nhóm. Ví dụ: đặt timezone cụ thể giúp mọi truy vấn datetime trả kết quả cùng múi giờ, bất kể cài đặt mặc định của server.
</Tip>

<Note>
Lệnh khởi động chạy mỗi lần kết nối, kể cả khi tự động kết nối lại. Lệnh phải đúng cú pháp của loại database: dùng cú pháp MySQL cho kết nối MySQL, PostgreSQL cho PostgreSQL, v.v.
</Note>

## Chỉnh sửa Kết nối

Sửa kết nối hiện có:

1. Chuột phải kết nối trong sidebar
2. Chọn **Edit Connection**
3. Thay đổi thông tin
4. Nhấp **Save**

<Warning>
Thay đổi có hiệu lực ở lần kết nối tiếp theo. Nếu đang kết nối, cần ngắt và kết nối lại.
</Warning>

## Xóa Kết nối

Xóa kết nối:

1. Chuột phải kết nối trong sidebar
2. Chọn **Delete Connection**
3. Xác nhận xóa

<Note>
Xóa kết nối chỉ xóa cài đặt và thông tin đăng nhập đã lưu. Database thực tế không bị ảnh hưởng.
</Note>

## Lưu trữ Kết nối

### Nơi lưu trữ

| Dữ liệu | Vị trí | Loại |
|------|----------|--------------|
| Cài đặt kết nối | `~/Library/Preferences/com.TablePro.plist` | UserDefaults |
| Cài đặt ứng dụng | `~/Library/Preferences/com.TablePro.plist` | UserDefaults |
| Mật khẩu | macOS Keychain | Mã hóa |
| SSH keys | Tham chiếu theo đường dẫn | Không sao chép |

### Sao lưu và Khôi phục

Sao lưu kết nối và cài đặt:

```bash
# Sao lưu
cp ~/Library/Preferences/com.TablePro.plist ~/Desktop/tablepro-backup.plist
```

Khôi phục:

```bash
# Đóng TablePro trước, rồi khôi phục
cp ~/Desktop/tablepro-backup.plist ~/Library/Preferences/com.TablePro.plist
```

<Warning>
File sao lưu không gồm mật khẩu (lưu trong Keychain) hay SSH private keys. Bạn cần nhập lại mật khẩu sau khi khôi phục.
</Warning>
