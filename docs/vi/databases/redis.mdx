# Kết nối Redis

TablePro hỗ trợ Redis 6.0 trở lên. Key được nhóm theo namespace phân cách bởi dấu hai chấm trong sidebar. Giá trị hiển thị với format theo kiểu dữ liệu (string, hash, list, set, sorted set, JSON).

## Thiết lập Nhanh

<Steps>
  <Step title="Mở Form Kết nối">
    Nhấp **New Connection** từ màn hình Chào mừng hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn Redis">
    Chọn **Redis** từ trình chọn loại database
  </Step>
  <Step title="Nhập Thông tin Kết nối">
    Điền host, port, password (nếu cần) và database index
  </Step>
  <Step title="Kiểm tra và Kết nối">
    Nhấp **Test Connection**, rồi **Create**
  </Step>
</Steps>

## Cài đặt Kết nối

### Trường Bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Tên định danh kết nối | - |
| **Host** | Hostname hoặc IP server | `localhost` |
| **Port** | Cổng Redis | `6379` |

### Trường Tùy chọn

| Trường | Mô tả |
|-------|-------------|
| **Password** | Mật khẩu Redis (AUTH) |
| **Database** | Chỉ số database (0-15, mặc định `0`) |

<Tip>
Cho dev local không bật xác thực, để trống password. Redis mặc định dùng database index `0`.
</Tip>

## Định dạng URL

Import kết nối bằng Redis URL.

Scheme `rediss://` (hai chữ `s`) chỉ kết nối TLS.

Xem [Tham chiếu URL kết nối](/databases/connection-urls#redis) để biết format đầy đủ.

## Ví dụ Cấu hình

### Server Dev Local

```
Name:     Local Redis
Host:     localhost
Port:     6379
Password: (để trống)
Database: 0
```

### Docker Redis Container

```
Name:     Docker Redis
Host:     localhost
Port:     6379 (hoặc cổng mapped)
Password: (mật khẩu từ REDIS_PASSWORD)
Database: 0
```

### Redis Cloud / Dịch vụ Quản lý

```
Name:     Redis Cloud
Host:     redis-12345.c1.us-east-1-2.ec2.cloud.redislabs.com
Port:     12345
Password: (mật khẩu Redis Cloud)
Database: 0
```

<Note>
Dịch vụ Redis quản lý thường bắt buộc SSL/TLS. Bật trong phần SSL/TLS của form kết nối.
</Note>

### Server Từ xa

```
Name:     Production Cache
Host:     redis.example.com
Port:     6379
Password: (mật khẩu bảo mật)
Database: 0
```

<Warning>
Cho Redis production, nên dùng [SSH tunneling](/vi/databases/ssh-tunneling) để kết nối an toàn.
</Warning>

## Kết nối SSL/TLS

Cấu hình SSL trong phần **SSL/TLS** của form kết nối.

| Chế độ SSL | Mô tả |
|----------|-------------|
| **Disabled** | Không mã hóa SSL |
| **Required** | Bắt buộc SSL |
| **Verify CA** | Bắt buộc SSL và xác minh chứng chỉ server với CA |

Với **Verify CA**, cung cấp file chứng chỉ CA. Tùy chọn thêm file chứng chỉ client và key cho mutual TLS.

<Tip>
Dùng scheme `rediss://` khi import kết nối yêu cầu TLS.
</Tip>

<Note>
Nếu không muốn cấu hình chứng chỉ SSL, [SSH tunneling](/vi/databases/ssh-tunneling) mã hóa toàn bộ traffic qua tunnel SSH.
</Note>

## Hỗ trợ SSH Tunnel

Kết nối Redis qua SSH tunnel để truy cập an toàn server từ xa. Xem [SSH Tunneling](/vi/databases/ssh-tunneling) để biết cách cài đặt.

## Tính năng

### Duyệt Namespace

Sau khi kết nối, sidebar nhóm key theo tiền tố dấu hai chấm. Key tên `user:1`, `user:2` và `session:abc` nằm trong namespace `user` và `session`:

1. Nhấp tên kết nối trong sidebar
2. Mở rộng namespace để xem key
3. Nhấp key để xem giá trị

Key không có dấu hai chấm nằm trong namespace gốc.

### Xem Key-Value

Giá trị hiển thị trong data grid theo kiểu dữ liệu:

- **String** hiển thị văn bản thuần
- **Hash** hiển thị cặp key-value theo hàng
- **List** hiển thị với chỉ số
- **Set** hiển thị từng hàng riêng
- **Sorted Set** hiển thị kèm điểm số
- **JSON** tô sáng cú pháp

### Quản lý TTL

Xem và quản lý thời gian sống (TTL) cho key:

- TTL hiển thị cạnh mỗi key trong data grid
- Key không hết hạn hiển thị `-1`
- Key không tồn tại hiển thị `-2`
- Đặt hoặc cập nhật TTL trực tiếp từ giao diện

### Redis CLI

Chạy lệnh Redis trực tiếp trong editor tab:

```redis
-- Thao tác key
GET mykey
SET mykey "hello" EX 60
DEL mykey key2 key3
KEYS user:*

-- Thao tác hash
HGETALL myhash
HSET myhash field1 "value1"
HDEL myhash field1

-- Thao tác list
LRANGE mylist 0 -1
LPUSH mylist "item1" "item2"
LLEN mylist

-- Thao tác set
SMEMBERS myset
SADD myset "member1" "member2"
SCARD myset

-- Thao tác sorted set
ZRANGE myzset 0 -1 WITHSCORES
ZADD myzset 1 "one" 2 "two"
ZCARD myzset

-- Quét key
SCAN 0 MATCH user:* COUNT 100

-- Thông tin server
PING
INFO
DBSIZE
```

### Lệnh Được Hỗ trợ

TablePro hỗ trợ đầy đủ tập lệnh Redis:

| Danh mục | Lệnh |
|----------|----------|
| **String** | GET, SET, MGET, MSET, INCR, DECR, APPEND |
| **Hash** | HGET, HSET, HGETALL, HDEL, HKEYS, HVALS, HLEN |
| **List** | LPUSH, RPUSH, LPOP, RPOP, LRANGE, LLEN, LINDEX |
| **Set** | SADD, SREM, SMEMBERS, SISMEMBER, SCARD, SUNION |
| **Sorted Set** | ZADD, ZREM, ZRANGE, ZRANGEBYSCORE, ZCARD, ZSCORE |
| **Key** | DEL, EXISTS, EXPIRE, TTL, TYPE, RENAME, KEYS, SCAN |
| **Server** | PING, INFO, DBSIZE, FLUSHDB, SELECT |

Lệnh không liệt kê ở trên vẫn gửi đến server và phản hồi hiển thị trong bảng kết quả.

## Khắc phục sự cố

### Kết nối Bị từ chối

**Triệu chứng**: "Connection refused" hoặc timeout

**Nguyên nhân và Giải pháp**:

1. **Redis không chạy**
   ```bash
   # Kiểm tra Redis
   redis-cli ping

   # Khởi động (macOS Homebrew)
   brew services start redis
   ```

2. **Sai cổng**
   - Kiểm tra cổng trong `redis.conf`

3. **Redis không lắng nghe đúng địa chỉ**
   - Kiểm tra `bind` trong `redis.conf`
   - Cho kết nối remote, thêm IP client hoặc dùng `0.0.0.0`

### Xác thực Thất bại

**Triệu chứng**: "NOAUTH Authentication required" hoặc "ERR invalid password"

**Giải pháp**:

1. Kiểm tra password khớp `requirepass` trong `redis.conf`
2. Cho Redis 6.0+ với ACL:
   ```redis
   ACL LIST
   ACL SETUSER myuser on >password ~* +@all
   ```

### Hết thời gian Kết nối

**Triệu chứng**: Kết nối treo hoặc timeout

**Giải pháp**:

1. Kiểm tra host và port đúng
2. Kiểm tra mạng và firewall
3. Cho Redis cloud, đảm bảo IP được whitelist

## Hạn chế

- **Cluster mode**: Redis Cluster sharding tự động không hỗ trợ. Kết nối trực tiếp từng node.
- **Pub/Sub**: Pub/Sub thời gian thực không hỗ trợ trong phiên bản hiện tại.
- **Streams**: Redis Streams (XADD, XREAD, v.v.) không hỗ trợ trong data grid, nhưng chạy được trong CLI.
- **Modules**: RedisJSON, RediSearch và module khác chạy qua CLI nhưng chưa có UI chuyên biệt.
- **Key lớn**: Key có hàng triệu phần tử được phân trang để tránh vấn đề bộ nhớ.

## Mẹo Hiệu suất

### Database Lớn

Cho Redis có hàng triệu key:

1. Dùng namespace browsing để lọc key theo tiền tố
2. Dùng `SCAN` thay `KEYS` trong CLI
3. Phân trang áp dụng tự động khi duyệt collection lớn

### Giám sát Bộ nhớ

Kiểm tra bộ nhớ Redis:

```redis
INFO memory
MEMORY USAGE mykey
DBSIZE
```
