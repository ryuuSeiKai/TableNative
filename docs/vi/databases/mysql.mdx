# Kết nối MySQL & MariaDB

TablePro hỗ trợ MySQL 5.7+, MySQL 8.0+ và MariaDB 10.x+. Cả hai dùng giao thức tương thích nên chia sẻ cùng giao diện kết nối.

## Thiết lập Nhanh

<Steps>
  <Step title="Mở Form Kết nối">
    Nhấp **New Connection** từ màn hình Chào mừng hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn Loại Database">
    Chọn **MySQL** hoặc **MariaDB** từ trình chọn loại
  </Step>
  <Step title="Nhập Thông tin Kết nối">
    Điền host, port, username, password và tùy chọn database mặc định
  </Step>
  <Step title="Kiểm tra và Kết nối">
    Nhấp **Test Connection**, rồi **Create**
  </Step>
</Steps>

## Cài đặt Kết nối

### Trường Bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Tên định danh kết nối | - |
| **Host** | Hostname hoặc IP server | `localhost` |
| **Port** | Cổng MySQL | `3306` |
| **Username** | Người dùng MySQL | `root` |
| **Password** | Mật khẩu | - |

### Trường Tùy chọn

| Trường | Mô tả |
|-------|-------------|
| **Database** | Database mặc định sau khi kết nối |

<Tip>
Để trống trường database sẽ hiển thị tất cả database mà user có quyền truy cập.
</Tip>

<Tip>
Bạn cũng có thể kết nối bằng cách mở URL `mysql://` hoặc `mariadb://` từ trình duyệt hoặc terminal. Xem [Tham chiếu URL kết nối](/databases/connection-urls).
</Tip>

## Ví dụ Cấu hình

### Server Dev Local

```
Name:     Local MySQL
Host:     localhost
Port:     3306
Username: root
Password: (mật khẩu)
Database: (để trống)
```

### Docker MySQL Container

```
Name:     Docker MySQL
Host:     localhost
Port:     3306 (hoặc cổng mapped)
Username: root
Password: (mật khẩu từ MYSQL_ROOT_PASSWORD)
Database: (để trống)
```

### MAMP Pro

```
Name:     MAMP MySQL
Host:     localhost
Port:     8889 (mặc định MAMP)
Username: root
Password: root
Database: (để trống)
```

### Server Từ xa

```
Name:     Production DB
Host:     db.example.com
Port:     3306
Username: app_user
Password: (mật khẩu bảo mật)
Database: production
```

<Warning>
Cho database production, nên dùng [SSH tunneling](/vi/databases/ssh-tunneling) để kết nối an toàn.
</Warning>

## MySQL so với MariaDB

TablePro dùng cùng driver cho cả hai. Một vài khác biệt:

| Tính năng | MySQL 8.0 | MariaDB 10.x |
|---------|-----------|--------------|
| Xác thực mặc định | `caching_sha2_password` | `mysql_native_password` |
| JSON | Native | Tương thích |
| Window functions | Có | Có |
| CTEs | Có | Có |

<Note>
Nếu gặp lỗi xác thực với MySQL 8.0, xem phần [Khắc phục Xác thực](#xac-thuc-mysql-80).
</Note>

### Thông tin Bảng

Mỗi bảng hiển thị:

- **Structure**: Định nghĩa cột, kiểu và ràng buộc
- **Indexes**: Tất cả index bao gồm primary keys
- **Foreign Keys**: Quan hệ với bảng khác
- **DDL**: Câu lệnh CREATE TABLE

### Thực thi Truy vấn

Hỗ trợ đầy đủ cú pháp MySQL:

```sql
-- Select với JSON
SELECT id, JSON_EXTRACT(data, '$.name') as name
FROM users
WHERE JSON_CONTAINS(roles, '"admin"');

-- Window functions
SELECT
    department,
    employee,
    salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) as rank
FROM employees;

-- CTEs
WITH RECURSIVE category_tree AS (
    SELECT id, name, parent_id, 0 as level
    FROM categories WHERE parent_id IS NULL
    UNION ALL
    SELECT c.id, c.name, c.parent_id, ct.level + 1
    FROM categories c
    JOIN category_tree ct ON c.parent_id = ct.id
)
SELECT * FROM category_tree;
```

## Khắc phục sự cố

### Kết nối Bị từ chối

**Triệu chứng**: "Connection refused" hoặc "Can't connect to MySQL server"

**Nguyên nhân và Giải pháp**:

1. **MySQL không chạy**
   ```bash
   # Kiểm tra MySQL
   mysqladmin ping -h localhost -u root -p

   # Khởi động MySQL (macOS Homebrew)
   brew services start mysql
   ```

2. **Sai cổng**
   - Xác minh cổng: `SHOW VARIABLES LIKE 'port';`
   - Kiểm tra file cấu hình

3. **MySQL không lắng nghe TCP**
   - Kiểm tra `skip-networking` không được đặt trong `my.cnf`
   - Đảm bảo `bind-address` bao gồm IP client

### Truy cập Bị từ chối

**Triệu chứng**: "Access denied for user 'xxx'@'localhost'"

**Giải pháp**:

1. Xác minh username và password
2. Kiểm tra quyền:
   ```sql
   SHOW GRANTS FOR 'username'@'localhost';
   ```
3. Cấp quyền:
   ```sql
   GRANT ALL PRIVILEGES ON database_name.* TO 'username'@'localhost';
   FLUSH PRIVILEGES;
   ```

### Xác thực MySQL 8.0

MySQL 8.0 mặc định dùng `caching_sha2_password`, có thể gây lỗi kết nối.

**Cách 1**: Dùng native password cho user cụ thể
```sql
ALTER USER 'username'@'localhost'
IDENTIFIED WITH mysql_native_password BY 'password';
```

**Cách 2**: Đổi phương thức xác thực mặc định
```ini
# Trong my.cnf
[mysqld]
default_authentication_plugin=mysql_native_password
```

### Kết nối SSL/TLS

Cấu hình SSL trong phần **SSL/TLS** của form kết nối.

| Chế độ SSL | Hành vi MySQL |
|----------|----------------|
| **Disabled** | SSL tắt hoàn toàn |
| **Preferred** | Dùng SSL nếu server hỗ trợ, không thì kết nối thường |
| **Required** | Bắt buộc SSL (`MYSQL_OPT_SSL_ENFORCE`) |
| **Verify CA** | Bắt buộc SSL và xác minh chứng chỉ server với CA |
| **Verify Identity** | Bắt buộc SSL, xác minh CA và hostname server |

Với **Verify CA** và **Verify Identity**, cung cấp đường dẫn file chứng chỉ CA. Tùy chọn thêm file chứng chỉ client và key cho mutual TLS.

<Tip>
Hầu hết nhà cung cấp MySQL cloud (AWS RDS, Google Cloud SQL, Azure) yêu cầu SSL. Dùng **Verify CA** với chứng chỉ CA của nhà cung cấp.
</Tip>

<Note>
Nếu không cần SSL nhưng muốn mã hóa, [SSH tunneling](/vi/databases/ssh-tunneling) mã hóa toàn bộ traffic qua tunnel SSH.
</Note>

## Mẹo Hiệu suất

### Tập Kết quả Lớn

Cho bảng hàng triệu dòng:

1. Dùng `LIMIT` trong truy vấn
2. Bật phân trang trong cài đặt data grid
3. Tạo index phù hợp cho truy vấn phổ biến

### Truy vấn Phức tạp

1. Dùng `EXPLAIN` để phân tích query plan:
   ```sql
   EXPLAIN SELECT * FROM users WHERE email LIKE '%@example.com';
   ```

2. Theo dõi thời gian thực thi trong bảng kết quả

## Bộ ký tự

TablePro mặc định UTF-8 (utf8mb4) cho kết nối MySQL, xử lý đúng ký tự quốc tế, emoji và ký hiệu đặc biệt.

Nếu gặp lỗi encoding, kiểm tra charset của bảng/cột:

```sql
SHOW CREATE TABLE table_name;
```
