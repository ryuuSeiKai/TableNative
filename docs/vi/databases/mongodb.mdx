# Kết nối MongoDB

TablePro hỗ trợ MongoDB 5.0 trở lên. Collection hiển thị như bảng trong sidebar. Document hiển thị với các trường là cột, đối tượng lồng nhau render dạng JSON.

## Thiết lập Nhanh

<Steps>
  <Step title="Mở Form Kết nối">
    Nhấp **New Connection** từ màn hình Chào mừng hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn MongoDB">
    Chọn **MongoDB** từ trình chọn loại database
  </Step>
  <Step title="Nhập Thông tin Kết nối">
    Điền host, port, authentication database, username và password
  </Step>
  <Step title="Kiểm tra và Kết nối">
    Nhấp **Test Connection**, rồi **Create**
  </Step>
</Steps>

## Cài đặt Kết nối

### Trường Bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Tên định danh kết nối | - |
| **Host** | Hostname hoặc IP server | `localhost` |
| **Port** | Cổng MongoDB | `27017` |
| **Database** | Tên database | - |

### Trường Tùy chọn

| Trường | Mô tả |
|-------|-------------|
| **Username** | Người dùng MongoDB |
| **Password** | Mật khẩu |
| **Auth Database** | Database xác thực (mặc định `admin`) |

<Tip>
Cho dev local không bật xác thực, để trống username và password.
</Tip>

### Cài đặt Nâng cao

Cấu hình read preference và write concern trong tab **Advanced**. Chỉ áp dụng cho MongoDB.

| Trường | Mô tả | Tùy chọn |
|-------|-------------|---------|
| **Read Preference** | Chọn thành viên replica set nhận thao tác đọc | Default, Primary, Primary Preferred, Secondary, Secondary Preferred, Nearest |
| **Write Concern** | Mức xác nhận cho thao tác ghi | Default, Majority, 1, 2, 3 |

<Tip>
Mặc định phù hợp hầu hết trường hợp. Dùng **Secondary Preferred** hoặc **Nearest** để phân tải đọc. Dùng **Majority** write concern cho độ bền cao hơn.
</Tip>

## Ví dụ Cấu hình

### Server Dev Local

```
Name:     Local MongoDB
Host:     localhost
Port:     27017
Username: (để trống)
Password: (để trống)
Database: myapp
```

### Docker MongoDB Container

```
Name:     Docker MongoDB
Host:     localhost
Port:     27017 (hoặc cổng mapped)
Username: root
Password: (mật khẩu từ MONGO_INITDB_ROOT_PASSWORD)
Database: admin
```

### MongoDB Atlas

```
Name:     Atlas Cluster
Host:     cluster0.abc123.mongodb.net
Port:     27017
Username: atlas_user
Password: (mật khẩu Atlas)
Database: myapp
```

<Note>
MongoDB Atlas bắt buộc SSL/TLS. Bật trong phần SSL/TLS của form kết nối.
</Note>

### Server Từ xa

```
Name:     Production DB
Host:     db.example.com
Port:     27017
Username: app_user
Password: (mật khẩu bảo mật)
Database: production
```

<Warning>
Cho database production, nên dùng [SSH tunneling](/vi/databases/ssh-tunneling) để kết nối an toàn.
</Warning>

## Kết nối SSL/TLS

Cấu hình SSL trong phần **SSL/TLS** của form kết nối.

| Chế độ SSL | Mô tả |
|----------|-------------|
| **Disabled** | Không mã hóa SSL |
| **Required** | Bắt buộc SSL |
| **Verify CA** | Bắt buộc SSL và xác minh chứng chỉ server với CA |

Với **Verify CA**, cung cấp file chứng chỉ CA. Tùy chọn thêm file chứng chỉ client và key cho mutual TLS.

<Tip>
MongoDB Atlas luôn yêu cầu SSL/TLS. Dùng **Required** hoặc **Verify CA** khi kết nối Atlas.
</Tip>

<Note>
Nếu không muốn cấu hình chứng chỉ SSL, [SSH tunneling](/vi/databases/ssh-tunneling) mã hóa toàn bộ traffic qua tunnel SSH.
</Note>

### Xem Document

Document hiển thị trong data grid với các trường là cột:

- **Trường top-level** là các cột riêng
- **Đối tượng lồng nhau** render dạng chuỗi JSON
- **Mảng** hiển thị dạng JSON array
- **ObjectId** hiển thị dạng chuỗi
- **Kiểu BSON** (Date, Decimal128, Binary, v.v.) render với format phù hợp

<Tip>
Schema được suy luận bằng cách lấy mẫu document trong collection. Cột phản ánh hợp nhất tất cả trường tìm thấy.
</Tip>

### Nhân bản Collection

Nhân bản collection có sẵn bao gồm document và index:

1. Mở hộp thoại **Create Table**
2. Nhấp **Duplicate** trên toolbar
3. Chọn collection nguồn
4. TablePro copy document bằng `aggregate` với `$out` và tạo lại index non-primary

Collection nhân bản tên `tenGoc_copy` (thêm số nếu trùng tên).

### Truy vấn MongoDB Shell

Thực thi truy vấn MQL trong editor:

```javascript
// Tìm document với bộ lọc
db.users.find({ age: { $gte: 18 }, active: true })

// Tìm với projection và sort
db.orders.find(
  { status: "completed" },
  { customerId: 1, total: 1, date: 1 }
).sort({ date: -1 }).limit(20)

// Aggregation pipeline
db.sales.aggregate([
  { $match: { date: { $gte: ISODate("2025-01-01") } } },
  { $group: { _id: "$product", totalSales: { $sum: "$amount" } } },
  { $sort: { totalSales: -1 } },
  { $limit: 10 }
])

// Đếm document
db.users.countDocuments({ role: "admin" })

// Giá trị duy nhất
db.products.distinct("category")
```

### Thao tác CRUD

#### Chèn Document

```javascript
// Chèn một document
db.users.insertOne({
  name: "John Doe",
  email: "john@example.com",
  age: 30,
  tags: ["developer", "admin"]
})

// Chèn nhiều document
db.users.insertMany([
  { name: "Alice", email: "alice@example.com" },
  { name: "Bob", email: "bob@example.com" }
])
```

#### Cập nhật Document

```javascript
// Cập nhật một document
db.users.updateOne(
  { email: "john@example.com" },
  { $set: { age: 31, updatedAt: new Date() } }
)

// Cập nhật nhiều document
db.users.updateMany(
  { active: false },
  { $set: { status: "inactive" } }
)
```

#### Xóa Document

```javascript
// Xóa một document
db.users.deleteOne({ email: "john@example.com" })

// Xóa nhiều document
db.users.deleteMany({ active: false, lastLogin: { $lt: ISODate("2024-01-01") } })
```

## Khắc phục sự cố

### Kết nối Bị từ chối

**Triệu chứng**: "Connection refused" hoặc "couldn't connect to server"

**Nguyên nhân và Giải pháp**:

1. **MongoDB không chạy**
   ```bash
   # Kiểm tra MongoDB
   mongosh --eval "db.runCommand({ ping: 1 })"

   # Khởi động (macOS Homebrew)
   brew services start mongodb-community
   ```

2. **Sai cổng**
   - Kiểm tra cổng trong `mongod.conf`

3. **MongoDB không lắng nghe đúng địa chỉ**
   - Kiểm tra `bindIp` trong `mongod.conf`
   - Cho kết nối remote, thêm IP client hoặc dùng `0.0.0.0`

### Xác thực Thất bại

**Triệu chứng**: "Authentication failed" hoặc "command find requires authentication"

**Giải pháp**:

1. Kiểm tra username, password và auth database
2. Kiểm tra roles:
   ```javascript
   db.getUsers()
   ```
3. Tạo hoặc cập nhật user:
   ```javascript
   db.createUser({
     user: "appuser",
     pwd: "password",
     roles: [{ role: "readWrite", db: "myapp" }]
   })
   ```

### Hết thời gian Kết nối

**Triệu chứng**: Kết nối treo hoặc timeout

**Giải pháp**:

1. Kiểm tra host và port đúng
2. Kiểm tra mạng và firewall
3. Cho MongoDB Atlas, đảm bảo IP được whitelist trong Network Access

## Hạn chế

- **Transaction**: Transaction đa document cần replica set hoặc sharded cluster. Standalone không hỗ trợ.
- **Suy luận schema**: MongoDB không có schema cố định, cấu trúc cột suy luận bằng lấy mẫu. Collection có document đa dạng có thể thiếu một số trường.
- **Capped collection**: Một số thao tác (update, delete) bị giới hạn trên capped collection.
- **GridFS**: Không duyệt trực tiếp qua collection browser.
- **Change streams**: Không hỗ trợ trong phiên bản hiện tại.

## Mẹo Hiệu suất

### Collection Lớn

Cho collection hàng triệu document:

1. Luôn dùng bộ lọc hoặc `limit()` trong truy vấn
2. Bật phân trang trong data grid
3. Tạo index phù hợp cho query pattern phổ biến

### Quản lý Index

Giám sát và tạo index:

```javascript
// Liệt kê index
db.users.getIndexes()

// Tạo index
db.users.createIndex({ email: 1 }, { unique: true })

// Index kết hợp
db.orders.createIndex({ customerId: 1, date: -1 })
```

### Phân tích Truy vấn

Dùng `explain()` để xem hiệu năng:

```javascript
db.users.find({ age: { $gte: 18 } }).explain("executionStats")
```
