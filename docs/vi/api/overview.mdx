## URL Cơ sở

```
https://api.tablepro.app/v1
```

## Endpoints

Ba endpoint quản lý vòng đời giấy phép:

| Phương thức | Endpoint | Mô tả |
| ----------- | -------- | ----- |
| `POST` | `/license/activate` | Kích hoạt giấy phép trên một máy |
| `POST` | `/license/validate` | Xác thực giấy phép đang hoạt động |
| `POST` | `/license/deactivate` | Hủy kích hoạt giấy phép khỏi một máy |

## Định dạng Yêu cầu

Tất cả yêu cầu phải là JSON với header `Content-Type: application/json`.

### Tham số Chung

| Tham số | Kiểu | Mô tả |
| ------- | ---- | ----- |
| `license_key` | `string` | Khóa giấy phép định dạng `XXXXX-XXXXX-XXXXX-XXXXX-XXXXX` (chữ và số) |
| `machine_id` | `string` | SHA-256 hash định danh máy (64 ký tự hex) |

## Định dạng Phản hồi

Phản hồi kích hoạt và xác thực thành công trả về dữ liệu giấy phép đã ký:

```json
{
  "data": {
    "email": "john@example.com",
    "expires_at": "2027-02-11T00:00:00+00:00",
    "issued_at": "2026-02-12T10:30:00+00:00",
    "license_key": "ABCDE-12345-FGHIJ-67890-KLMNO",
    "status": "active"
  },
  "signature": "base64-encoded-signature"
}
```

Trường `signature` là chữ ký RSA-SHA256 của đối tượng `data`. Xác minh phía client bằng public key để đảm bảo tính toàn vẹn.

## Ký Phản hồi

Mọi phản hồi kích hoạt và xác thực thành công đều được ký mật mã:

1. Các trường `data` sắp xếp theo bảng chữ cái
2. Đối tượng đã sắp xếp được mã hóa JSON
3. Chuỗi JSON được ký bằng RSA-SHA256
4. Chữ ký trả về dạng Base64

Client có thể xác minh dữ liệu giấy phép chưa bị giả mạo.

## Giới hạn Tốc độ

Tất cả endpoint đều bị giới hạn. Thông tin trong header phản hồi:

| Header | Mô tả |
| ------ | ----- |
| `X-RateLimit-Limit` | Số yêu cầu tối đa trong cửa sổ hiện tại |
| `X-RateLimit-Remaining` | Số yêu cầu còn lại |
| `X-RateLimit-Reset` | Unix timestamp khi cửa sổ reset |

Khi vượt quá, API trả về `429`:

```json
{
  "message": "Too many requests.",
  "retry_after": 60
}
```

## Xử lý Lỗi

Phản hồi lỗi theo định dạng nhất quán:

```json
{
  "message": "Human-readable error message."
}
```

### Mã Trạng thái

| Mã | Mô tả |
| -- | ----- |
| `200` | Thành công |
| `403` | Giấy phép bị đình chỉ, hết hạn, hoặc máy chưa kích hoạt |
| `404` | Không tìm thấy khóa giấy phép |
| `409` | Đạt giới hạn kích hoạt |
| `422` | Lỗi xác thực (tham số không hợp lệ) |
| `429` | Vượt quá giới hạn tốc độ |

### Lỗi Xác thực

Phản hồi `422` có chi tiết theo trường:

```json
{
  "message": "The given data was invalid.",
  "errors": {
    "license_key": ["The license key field is required."],
    "machine_id": ["The machine id must be 64 characters."]
  }
}
```

## Trạng thái Giấy phép

| Trạng thái | Mô tả |
| ---------- | ----- |
| `active` | Giấy phép hợp lệ và dùng được |
| `suspended` | Giấy phép bị đình chỉ, không thể kích hoạt hay xác thực |
