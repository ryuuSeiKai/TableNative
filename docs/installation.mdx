# Installation

## System Requirements

### Minimum Requirements

| Requirement | Specification |
|-------------|---------------|
| **macOS Version** | macOS 14.0 (Sonoma) or later |
| **Processor** | Apple Silicon (M1/M2/M3/M4/M5) or Intel x86_64 |
| **Memory** | 4 GB RAM minimum |
| **Storage** | ~200 MB free disk space |

### Recommended

- **macOS**: Latest version for best performance
- **Memory**: 8 GB RAM or more for large databases
- **Storage**: SSD for faster query history and tab persistence

<Note>
TablePro uses native Apple frameworks only. No Java, .NET, or other runtimes needed.
</Note>

## Install via Homebrew

The fastest option:

```bash
brew install --cask tablepro
```

<Tip>
The cask formula automatically removes the macOS quarantine attribute, so you skip the `xattr -cr` step on first launch.
</Tip>

To update:

```bash
brew upgrade tablepro
```

To uninstall:

```bash
brew uninstall tablepro
```

To also remove application data:

```bash
brew zap tablepro
```

## Download

### From GitHub Releases

1. Go to the [TablePro Releases](https://github.com/datlechin/tablepro/releases) page
2. Find the latest release
3. Download the right DMG for your Mac:

| Architecture | File | For Macs with |
|--------------|------|---------------|
| Apple Silicon | `TablePro-arm64.dmg` | M1, M2, M3, M4, M5 chips |
| Intel | `TablePro-x86_64.dmg` | Intel processors |

### Checking Your Mac's Architecture

1. Click the **Apple menu** () in the menu bar
2. Select **About This Mac**
3. Look for:
   - **Chip**: Apple M1/M2/M3/M4/M5 = Apple Silicon
   - **Processor**: Intel = x86_64

## Installation Steps

### Standard Installation

1. **Open the DMG file**
   - Double-click the downloaded `.dmg` file
   - A new Finder window appears

2. **Drag to Applications**
   - Drag the TablePro icon to the Applications folder shortcut
   - Wait for the copy to complete

### First Launch Security

TablePro is not signed with an Apple Developer certificate. macOS will show a **"TablePro is damaged and can't be opened"** error on first launch. This is the standard quarantine behavior for unsigned apps.

<Warning>
This is normal for unsigned applications downloaded from the internet. TablePro is source-available, so the code is publicly viewable for inspection.
</Warning>

#### Remove the quarantine attribute

Open **Terminal** and run:

```bash
xattr -cr /Applications/TablePro.app
```

Then open TablePro normally from Applications.

<Tip>
One-time step. After removing the quarantine attribute, TablePro opens normally on all future launches.
</Tip>

#### Alternative: System Settings

If the security warning persists after running the command:

1. Open **System Settings** > **Privacy & Security**
2. Scroll down to the **Security** section
3. You should see a message about TablePro being blocked
4. Click **Open Anyway**
5. Enter your password if prompted

## Updating TablePro

### Automatic Updates

TablePro checks for updates in the background via [Sparkle](https://sparkle-project.org/) and notifies you when a new version is available.

### Manual Update Check

To check manually:

1. Open **TablePro** menu in the menu bar
2. Click **Check for Updates...**

### Update Settings

Configure update behavior in **Settings** > **General** > **Software Update**:

- Enable or disable automatic update checks
- Set the check frequency

<Tip>
Connections and settings are stored separately from the app binary and persist across updates.
</Tip>

### Data Storage Locations

| Data | Location | Storage Type |
|------|----------|--------------|
| **Connections** | `~/Library/Preferences/com.TablePro.plist` | UserDefaults |
| **Settings** | `~/Library/Preferences/com.TablePro.plist` | UserDefaults |
| **Passwords** | macOS Keychain (secure storage) | Keychain |
| **Query History** | `~/Library/Application Support/TablePro/query_history.db` | SQLite |

## Uninstallation

### Remove the Application

1. Quit TablePro if it's running
2. Open **Finder** > **Applications**
3. Drag **TablePro** to the Trash

### Remove Application Data (Optional)

To delete all TablePro data:

```bash
# Remove application support data
rm -rf ~/Library/Application\ Support/TablePro

# Remove preferences
rm ~/Library/Preferences/com.TablePro.plist

# Remove Keychain items (via Keychain Access app)
# Search for "TablePro" and delete related items
```

<Warning>
This deletes all saved connections and query history. Export anything important first.
</Warning>

## Troubleshooting

### "TablePro is damaged and can't be opened"

Follow the [First Launch Security](#first-launch-security) instructions to remove the macOS quarantine attribute.

### "TablePro can't be opened because it is from an unidentified developer"

Follow the [First Launch Security](#first-launch-security) instructions.

### Application crashes on launch

1. Check that you downloaded the correct architecture version
2. Verify you're running macOS 14.0 or later
3. Try removing and reinstalling the application
4. Check Console.app for crash logs

### Connection issues after installation

If TablePro opens but can't connect to databases:

1. Verify your database server is running
2. Check firewall settings (System Settings > Network > Firewall)
3. Make sure the database server accepts connections from localhost

## Building from Source

For developers who want to build TablePro from source:

1. Clone the repository:
```bash
git clone https://github.com/datlechin/tablepro.git
cd tablepro
```

2. Open the Xcode project:
```bash
open TablePro.xcodeproj
```

3. Build and run:
   - Select the **TablePro** scheme
   - Press `Cmd+R` to build and run

See the [Development Setup](/development/setup) guide for detailed instructions.
